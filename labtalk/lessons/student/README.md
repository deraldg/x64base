# Student Lessons

Student lessons are learner-facing or learner-candidate material. They should be
written so a student can tell what to do, what to observe, what is proven, and
what is still under review.

Current seed lessons:

- `records_fields_tables_v0.md`
- `comments_to_contracts_to_help_v0.md`
- `data_systems_history_trail_v0.md`
- `schema_change_real_world_field_types_v0.md`

Promotion rule:

```text
draft -> proof_linked -> student_ready_pending -> student_ready
```

The final step requires reviewed setup notes, safe data handling, expected
observations, and proof links.
