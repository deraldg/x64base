title: "WSL and wx launch lanes documented"
description: "The project now carries clearer launch and build guidance for the Linux build path and the native wx workbench lane."
date: "2026-07-08"
---

Cross-platform and GUI work are now being documented with less guesswork.

That includes:

- clearer WSL build expectations
- explicit wx workbench launch paths
- Python preview lane visibility as a parallel GUI/prototyping surface
- a stronger rule that front ends consume DotTalk++ and x64base truth instead of replacing it

This is part of the broader teaching goal: students should be able to see how the same engine supports multiple interfaces without forking the architecture into unrelated products.
