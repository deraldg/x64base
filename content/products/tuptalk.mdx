## Overview

TupTalk is best described today as the tuple-facing part of the DotTalk++
inspection model, not a separately finished standalone product.

Its subject is the row as a tuple: current record display, projected row output,
tuple export, tuple validation, and relation-derived tuple grids.

The local command surface includes tuple and browser-related commands such as:

- `SIMPLEBROWSE` / `SB`
- `SMARTBROWSE` / `SM` / `SMART`
- `SMARTLIST`
- `TUPLE`
- `TUPTALK`
- `TUPEXPORT`
- `TUPVALIDATE`
- relation-aware `ERSATZ` grid output

## Typical tasks

- Inspect current rows as tuples instead of raw table internals.
- Review projected tuple-style rows.
- Explore relation-derived row output.
- Validate tuple/export behavior while keeping table mutation separate.

See [TupTalk docs](/docs/talk-family/tuptalk), [Current Project Truth](/docs/dev/project-truth), and the canonical public repo [deraldg/x64base](https://github.com/deraldg/x64base).
