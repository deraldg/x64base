## Overview

LabTalk® is the learning-lab and campus layer for x64base and DotTalk++.

![LabTalk Laboratory Campus](/images/labtalk/labtalk_campus_map_v0.svg)

It presents the system as a set of laboratories:

- runtime systems
- historical data systems from punch cards to cloud computing and AI
- self-documenting systems
- datasets and cases
- GUI and portal experiments
- proof-backed lessons

## What It Is

LabTalk is a way to organize learning around live systems:

![LabTalk proof-aware learning path](/images/labtalk/labtalk_proof_path_v0.svg)

```text
concept -> app -> dataset -> command -> proof -> case -> lesson
```

That pattern lets a learner see not only what a database command does, but where
the explanation came from, what data was used, what proof exists, and how the
idea fits into computing history.

The [Database Evolution Path](/docs/labtalk/database-evolution) integrates the
LabTalk / DotTalk++ storyboard deck as a public teaching path and local
source artifact.

The [Education Features](/docs/labtalk/education-features) page connects LabTalk
to source-defined teaching mechanisms: timed `IDX` sort/index labs, student
command and function extension hooks, pre/post polling hooks, and source-derived
command/function catalogs.

## Current Position

LabTalk should be described as an educational/laboratory campus direction, not
as a finished packaged distribution unless that package has been explicitly
released.
