## Overview

The **Laboratory Campus** is the learning and collaboration layer for x64base
and DotTalk++. LabTalk® remains the project mark and legacy name, but public
copy should increasingly emphasize the campus.

![LabTalk Laboratory Campus](/images/labtalk/labtalk_campus_map_v0.svg)

It presents the system as a set of laboratories:

- runtime systems
- historical data systems from punch cards to cloud computing and AI
- self-documenting systems
- datasets and cases
- GUI and portal experiments
- proof-backed lessons

## What It Is

The Laboratory Campus is a way to organize learning around live systems:

![LabTalk proof-aware learning path](/images/labtalk/labtalk_proof_path_v0.svg)

```text
concept -> app -> dataset -> command -> proof -> case -> lesson
```

That pattern lets a learner see not only what a database command does, but where
the explanation came from, what data was used, what proof exists, and how the
idea fits into computing history.

The [Database Evolution Path](/docs/labtalk/database-evolution) integrates the
Laboratory Campus / DotTalk++ storyboard deck as a public teaching path and local
source artifact.

The [Education Features](/docs/labtalk/education-features) page connects the
campus to source-defined teaching mechanisms: timed `IDX` sort/index labs,
student command and function extension hooks, pre/post polling hooks, and
source-derived command/function catalogs.

## Current Position

The Laboratory Campus is **alpha**. It is a hybrid of planning material,
implemented pieces, and in-development collaboration areas. Public pages should
describe it as an educational campus direction and collaboration frame, not as a
finished packaged distribution unless that package has been explicitly released.
