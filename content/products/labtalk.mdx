## Overview

The **Laboratory Campus** is the learning and collaboration layer for x64base
and DotTalk++. LabTalk® remains the project mark and legacy name, but public
copy should increasingly emphasize the campus.

[![Laboratory Campus visual](/images/labtalk/laboratory-campus-map-v2.svg)](/docs/labtalk/cases-storyboard)

## Cases and Storyboard

The [Cases and Storyboard](/docs/labtalk/cases-storyboard) page is the visible
entry point for the LabTalk / DotTalk++ systems storyboard deck, database-history
case families, source-memory stories, and proof-aware classroom paths. It should
not be buried inside a single evolution page because it explains why the campus
connects personal experience, historical data systems, live runtime evidence,
and lessons.

It presents the system as a set of laboratories:

- runtime systems
- historical data systems from punch cards to cloud computing and AI
- self-documenting systems
- datasets and cases
- GUI, pycrud, and portal experiments
- proof-backed lessons

## What It Is

The Laboratory Campus is a way to organize learning around live systems:

[![Laboratory Campus proof-aware learning path](/images/labtalk/laboratory-campus-proof-path-v2.svg)](/docs/labtalk/cases-storyboard)

```text
concept -> app -> dataset -> command -> proof -> case -> lesson
```

That pattern lets a learner see not only what a database command does, but where
the explanation came from, what data was used, what proof exists, and how the
idea fits into computing history.

The [Database Evolution Path](/docs/labtalk/database-evolution) integrates the
Laboratory Campus / DotTalk++ storyboard deck as a public teaching path and local
source artifact.

The [Education Features](/docs/labtalk/education-features) page connects the
campus to source-defined teaching mechanisms: timed `IDX` sort/index labs,
student command and function extension hooks, pre/post polling hooks, and
source-derived command/function catalogs.

The [Lesson Platform](/docs/labtalk/lessons) curates student lesson seeds,
career lessons learned while building the system, and structured lesson
suggestions from educators, students, technical writers, and collaborators.

The local LabTalk portal also exposes **pycrud** as a launcher-backed campus
tool. pycrud is a small Python/Tk CRUD and relational teaching surface with a
local FastAPI bridge (`pydottalk_api`) for schema, fixtures, header metadata,
and table readback. It belongs in LabTalk as a practical GUI/database literacy
lab, not as the DotTalk++ engine itself.

## AI Portal — Alpha/Experimental

The [AI Portal](/docs/labtalk/ai-portal) is the proposed AI-facing campus
entrance. It aims to prepare an AI for a series of tasks through curated, typed
jumps, authority and proof checks, bounded context packets, and guarded action
planning.

The lane is **Alpha/Experimental**. Current onboarding documents, registries,
launcher surfaces, and `MAINT AI` readback are seeds—not production autonomous
memory and not a replacement for runtime truth or human authorization.

## Current Position

The Laboratory Campus is **alpha**. It is a hybrid of planning material,
implemented pieces, and in-development collaboration areas. Public pages should
describe it as an educational campus direction and collaboration frame, not as a
finished packaged distribution unless that package has been explicitly released.
