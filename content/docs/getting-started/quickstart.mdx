## Clone, build, and test

```powershell
git clone https://github.com/deraldg/x64base.git
cd x64base
cmake --preset windows-core
cmake --build --preset windows-core --parallel
ctest --preset windows-core
```

On Linux, WSL, or macOS, use `core-vcpkg` in place of `windows-core`.

## First command sequence

Run the built `dottalkpp` executable from a directory containing a disposable
copy of the sample data, then try:

```text
USE students
STRUCT
LIST NEXT 5
SET ORDER TO TAG lname
SEEK "WHITE"
SMARTLIST NEXT 5
```

Use `HELP <command>` for the current command contract. Mutation commands such
as `REPLACE`, `CALCWRITE`, `DELETE`, `COMMIT`, and `PACK` should be exercised
only against disposable data.

## Reality check

- `main` is the canonical public source.
- A green CI run proves only the selected build/test profiles.
- LMDB index commands, TUI, GUI, Python, education, and SelfDoc lanes require their own evidence; the LMDB library itself is a baseline message-catalog dependency.
- `COMMIT` is a best-effort buffer apply operation, not an atomic transaction
  across DBF, memo, and index stores.
