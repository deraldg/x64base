## What it does

TurboTalk names repeatable maintenance and automation workflows around the DotTalk++ command shell.

Real local command surfaces and workflow anchors include:

```text
BUILDLMDB
BUILDLMDB CLEAN YES
BUILDLMDB FORCE YES
REINDEX
CMDHELPCHK
CMDHELPCHK REF
WORKSPACE OPEN DBF
WORKSPACE OPEN DBF CDX
WORKSPACE SAVE <name>
WORKSPACE LOAD <name>
```

The local source also includes DotScript scripts, smoke checks, maintenance lanes, and generated proposal packages. Treat this as active workflow infrastructure, not as a separate finished batch product.
