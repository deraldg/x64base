9:T693,<h2 id="overview"><a class="no-underline" href="#overview">Overview</a></h2>
<p>x64base is the database/runtime side of the DotTalk++ system. In the local source tree it is best described as active beta engine work, not a finished standalone commercial product.</p>
<p>The current engine reality includes:</p>
<ul>
<li>DBF-style table handling.</li>
<li>x32 and x64-family table work.</li>
<li>field typing, validation, cursor state, record movement, and work areas.</li>
<li>memo subsystem work through <code>MemoRef</code>, <code>MemoManager</code>, <code>MemoObject</code>, and <code>MemoStore</code> concepts.</li>
<li>INX, CNX, CDX, and LMDB-backed index work.</li>
<li>data dictionary and metadata catalog readers.</li>
<li>command-shell integration through DotTalk++.</li>
</ul>
<h2 id="what-is-proven-versus-active"><a class="no-underline" href="#what-is-proven-versus-active">What is proven versus active</a></h2>
<p>Use "runtime-proven" only when a smoke path or command surface has current evidence. Use "active", "partial", "experimental", or "canary" where the local project says the feature is still being integrated.</p>
<p>Current active areas include CDX/LMDB lifecycle work, memo backend proof, SYS* metadata congruence, and GUI/runtime bridging.</p>
