2:Tcee,<h2 id="overview"><a class="no-underline" href="#overview">Overview</a></h2>
<p>DotTalk++ is not just an educational scripting language. In the local project it is the canonical runtime surface for x64base work.</p>
<p>It owns or exposes:</p>
<ul>
<li>interactive commands and DotScript automation.</li>
<li>table opening, work areas, cursor movement, record inspection, and mutation commands.</li>
<li>HELP, command reflection, function references, and <code>CMDHELPCHK</code>.</li>
<li>relation-aware browsing through <code>ERSATZ</code> / <code>RBROWSE</code>.</li>
<li>table listing and browsing through <code>SMARTLIST</code>, <code>SMARTBROWSE</code>, <code>SM</code>, <code>SMART</code>, <code>SIMPLEBROWSE</code>, <code>SB</code>, <code>BROWSE</code>, <code>BROWSER</code>, and <code>BROWSETUI</code>.</li>
<li>workspace save/load behavior and relation state.</li>
<li>metadata validation, data dictionary work, and generated reports.</li>
<li>TUI, wxWidgets, and Python/Tk GUI lanes over the same runtime concepts.</li>
</ul>
<h2 id="dotscript-relationship"><a class="no-underline" href="#dotscript-relationship">DotScript relationship</a></h2>
<p>DotTalk++ is the interactive shell and canonical command runtime. DotScript is the productized script language built from the same command surface. DotScript covers command files, variables, comments, line continuation, <code>IF</code> / <code>ELSE</code> / <code>ENDIF</code>, <code>LOOP</code> / <code>ENDLOOP</code>, <code>WHILE</code> / <code>ENDWHILE</code>, <code>UNTIL</code> / <code>ENDUNTIL</code>, <code>SCAN</code> / <code>ENDSCAN</code>, one-level subscript nesting, CSV import/export workflows, and repeatable automation.</p>
<p>See <a href="/products/dotscript">DotScript</a> and <a href="/docs/dottalk/dotscript-language-guide">DotScript Language Guide</a>.</p>
<h2 id="status"><a class="no-underline" href="#status">Status</a></h2>
<p>DotTalk++ is active beta. Some areas are runtime-proven and others are canaries or integration surfaces. Site copy should preserve those boundaries instead of flattening everything into "complete".</p>
<h2 id="runtime-footprint"><a class="no-underline" href="#runtime-footprint">Runtime footprint</a></h2>
<p>The current Windows release build is intentionally more than a thin launcher.
<code>dottalkpp.exe</code> carries the command shell, DBF runtime surface, scripting,
SelfDoc/HELP, metadata, workspace, relation, browser, and TUI hooks. SQLite and
LMDB are kept as separate DLL dependencies rather than being folded into the
executable.</p>
<p>See <a href="/docs/engine/runtime-footprint">Runtime Footprint and Dependencies</a>.</p>
