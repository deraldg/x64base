3:T4b5,<h2 id="overview"><a class="no-underline" href="#overview">Overview</a></h2>
<p>ArcticTalk is the code name for the DotTalk++ Workbench TUI surface.</p>
<p>The family group is <strong>DotTalk++ Workbench</strong>. ArcticTalk should not be used as
the umbrella name for every GUI, browser, shell, and workflow surface.</p>
<p>Its subject is the terminal/TUI layer around the runtime, not a claim that every
Workbench surface is ArcticTalk.</p>
<p>In the local source tree, the real surfaces include:</p>
<ul>
<li>ArcticTalk/Foxtalk TUI and shell-bridge work where the TUI is the active surface.</li>
<li>TUI tuple grids, record popups, command panes, and menu-driven runtime inspection.</li>
<li>Workbench evidence where the visible runtime identifies the Arctic/TUI code-name surface.</li>
</ul>
<p>Use DotTalk++ Workbench for the broader family. Use ArcticTalk for the TUI code
name.</p>
