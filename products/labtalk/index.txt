9:Ta21,<h2 id="overview"><a class="no-underline" href="#overview">Overview</a></h2>
<p>LabTalk® is the learning-lab and campus layer for x64base and DotTalk++.</p>
<p><img src="/images/labtalk/labtalk_campus_map_v0.svg" alt="LabTalk Laboratory Campus"></p>
<p>It presents the system as a set of laboratories:</p>
<ul>
<li>runtime systems</li>
<li>historical data systems from punch cards to cloud computing and AI</li>
<li>self-documenting systems</li>
<li>datasets and cases</li>
<li>GUI and portal experiments</li>
<li>proof-backed lessons</li>
</ul>
<h2 id="what-it-is"><a class="no-underline" href="#what-it-is">What It Is</a></h2>
<p>LabTalk is a way to organize learning around live systems:</p>
<p><img src="/images/labtalk/labtalk_proof_path_v0.svg" alt="LabTalk proof-aware learning path"></p>
<pre><code class="language-text">concept -> app -> dataset -> command -> proof -> case -> lesson
</code></pre>
<p>That pattern lets a learner see not only what a database command does, but where
the explanation came from, what data was used, what proof exists, and how the
idea fits into computing history.</p>
<p>The <a href="/docs/labtalk/database-evolution">Database Evolution Path</a> integrates the
LabTalk / DotTalk++ storyboard deck as a public teaching path and local
source artifact.</p>
<p>The <a href="/docs/labtalk/education-features">Education Features</a> page connects LabTalk
to source-defined teaching mechanisms: timed <code>IDX</code> sort/index labs, student
command and function extension hooks, pre/post polling hooks, and source-derived
command/function catalogs.</p>
<h2 id="current-position"><a class="no-underline" href="#current-position">Current Position</a></h2>
<p>LabTalk should be described as an educational/laboratory campus direction, not
as a finished packaged distribution unless that package has been explicitly
released.</p>
