2:T147b,<h2 id="overview"><a class="no-underline" href="#overview">Overview</a></h2>
<p>The <strong>Laboratory Campus</strong> is the learning and collaboration layer for x64base
and DotTalk++. LabTalk™ remains the project mark and legacy name, but public
copy should increasingly emphasize the campus.</p>
<p><a href="/docs/labtalk/cases-storyboard"><img src="/images/labtalk/laboratory-campus-live-systems.svg" alt="Learning around live systems in the Laboratory Campus"></a></p>
<h2 id="cases-and-storyboard"><a class="no-underline" href="#cases-and-storyboard">Cases and Storyboard</a></h2>
<p>The <a href="/docs/labtalk/cases-storyboard">Cases and Storyboard</a> page is the visible
entry point for the LabTalk / DotTalk++ systems storyboard deck, database-history
case families, source-memory stories, and proof-aware classroom paths. It should
not be buried inside a single evolution page because it explains why the campus
connects personal experience, historical data systems, live runtime evidence,
and lessons.</p>
<p>It presents the system as a set of laboratories:</p>
<ul>
<li>runtime systems</li>
<li>historical data systems from punch cards to cloud computing and AI</li>
<li>self-documenting systems</li>
<li>datasets and cases</li>
<li>GUI, pycrud, and portal experiments</li>
<li>proof-backed lessons</li>
</ul>
<h2 id="what-it-is"><a class="no-underline" href="#what-it-is">What It Is</a></h2>
<p>The Laboratory Campus is a way to organize learning around live systems:</p>
<p><a href="/docs/labtalk/cases-storyboard"><img src="/images/labtalk/laboratory-campus-proof-path-v2.svg" alt="Laboratory Campus proof-aware learning path"></a></p>
<pre><code class="language-text">concept -> app -> dataset -> command -> proof -> case -> lesson
</code></pre>
<p>That pattern lets a learner see not only what a database command does, but where
the explanation came from, what data was used, what proof exists, and how the
idea fits into computing history.</p>
<p>The <a href="/docs/labtalk/database-evolution">Database Evolution Path</a> integrates the
Laboratory Campus / DotTalk++ storyboard deck as a public teaching path and local
source artifact.</p>
<p>The <a href="/docs/labtalk/education-features">Education Features</a> page connects the
campus to source-defined teaching mechanisms: timed <code>IDX</code> sort/index labs,
student command and function extension hooks, pre/post polling hooks, and
source-derived command/function catalogs.</p>
<p>The <a href="/docs/labtalk/lessons">Lesson Platform</a> curates student lesson seeds,
career lessons learned while building the system, and structured lesson
suggestions from educators, students, technical writers, and collaborators.</p>
<p>The local LabTalk portal also exposes <strong>pycrud</strong> as a launcher-backed campus
tool. pycrud is a small Python/Tk CRUD and relational teaching surface with a
local FastAPI bridge (<code>pydottalk_api</code>) for schema, fixtures, header metadata,
and table readback. It belongs in LabTalk as a practical GUI/database literacy
lab, not as the DotTalk++ engine itself.</p>
<h2 id="ai-portal--alphaexperimental"><a class="no-underline" href="#ai-portal--alphaexperimental">AI Portal — Alpha/Experimental</a></h2>
<p>The <a href="/docs/labtalk/ai-portal">AI Portal</a> is the proposed AI-facing campus
entrance. It aims to prepare an AI for a series of tasks through curated, typed
jumps, authority and proof checks, bounded context packets, and guarded action
planning.</p>
<p>The lane is <strong>Alpha/Experimental</strong>. Current onboarding documents, registries,
launcher surfaces, and <code>MAINT AI</code> readback are seeds—not production autonomous
memory and not a replacement for runtime truth or human authorization.</p>
<h2 id="current-position"><a class="no-underline" href="#current-position">Current Position</a></h2>
<p>The Laboratory Campus is <strong>alpha</strong>. It is a hybrid of planning material,
implemented pieces, and in-development collaboration areas. Public pages should
describe it as an educational campus direction and collaboration frame, not as a
finished packaged distribution unless that package has been explicitly released.</p>
