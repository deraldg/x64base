15:T898,<h2 id="overview"><a class="no-underline" href="#overview">Overview</a></h2>
<p>DotScript is the script-language product that sits on top of the DotTalk++ command surface.</p>
<p>DotTalk++ is the interactive shell and runtime vocabulary. DotScript is the repeatable file and automation form: command sequences, variables, continuations, comments, conditionals, loops, scans, CSV/DBF import-export runs, validation runs, lab scripts, and reproducible database maintenance.</p>
<h2 id="product-role"><a class="no-underline" href="#product-role">Product role</a></h2>
<p>DotScript should be treated as a first-class product because it is the bridge between teaching, documentation, and repeatable operation.</p>
<p>It supports these work patterns:</p>
<ul>
<li>reusable command files for classroom and lab work.</li>
<li>database inspection and validation runs.</li>
<li>scripted table, workspace, memo, relation, and index exercises.</li>
<li>smoke tests and canary workflows that document behavior while they run.</li>
<li>examples that can be copied directly into student or developer sessions.</li>
</ul>
<h2 id="language-guide"><a class="no-underline" href="#language-guide">Language guide</a></h2>
<p>The public language guide lives at <a href="/docs/dottalk/dotscript-language-guide">DotScript Language Guide</a>.</p>
<p>That guide tracks the core syntax subjects that must not be lost:</p>
<ul>
<li><code>IF</code> / <code>ELSE</code> / <code>ENDIF</code>.</li>
<li><code>LOOP</code> / <code>ENDLOOP</code>.</li>
<li><code>WHILE</code> / <code>ENDWHILE</code>.</li>
<li><code>UNTIL</code> / <code>ENDUNTIL</code>.</li>
<li><code>SCAN</code> / <code>ENDSCAN</code>.</li>
<li>variables and assignment.</li>
<li>line continuation.</li>
<li>comments.</li>
<li>command files and one-level subscript examples.</li>
<li>CSV import/export to and from DBF workflows.</li>
</ul>
<h2 id="status"><a class="no-underline" href="#status">Status</a></h2>
<p>DotScript is active beta and should remain proof-labeled. Some commands are runtime-proven; others are canaries or planned integration surfaces.
