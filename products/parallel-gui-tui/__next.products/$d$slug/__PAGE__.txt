2:T5c1,<h2 id="overview"><a class="no-underline" href="#overview">Overview</a></h2>
<p>Parallel GUI/TUI is the DotTalk++ Workbench product lane for keeping graphical
and terminal interfaces aligned over the same x64base and DotTalk++ runtime.</p>
<p>The goal is a shared engine truth: the GUI, TUI, CLI, browser, and workbench
surfaces should report the same work areas, table-buffer state, relation graph,
tuple output, command results, help metadata, and diagnostics.</p>
<p>Arctic is the code name for the TUI version. Arctic is not the family group and
is not positioned as one of the main products.</p>
<h2 id="scope"><a class="no-underline" href="#scope">Scope</a></h2>
<ul>
<li>DotTalk++ Workbench GUI and preview surfaces.</li>
<li>Arctic TUI menus, panes, tuple grids, and record popups.</li>
<li>CLI evidence that proves the same commands and runtime state outside the GUI.</li>
<li>Shared command routing, HELP/SelfDoc metadata, workspaces, relations, table
buffers, and output readback.</li>
</ul>
<h2 id="current-status"><a class="no-underline" href="#current-status">Current Status</a></h2>
<p>This is an active beta lane. Runtime screenshots show real GUI, TUI, and CLI
states, but they are evidence artifacts, not final release-polish claims.</p>
