6:Tffa,<p>Pinocchio is the disposable engine stress-test lane used to answer a practical
question: can the educational database behave like a real engine at
million-row scale? Its generated data, tables, indexes, and LMDB stores remain
isolated from protected project data.</p>
<h2 id="current-proof-state"><a class="no-underline" href="#current-proof-state">Current proof state</a></h2>
<p>The retained Phase 1 workload contains 1,000,000 student rows and 5,501,358
enrollment rows. The original read battery found that ordered <code>TOP</code> navigation
was scanning and sorting the table even when a CDX/LMDB order was active. A
development correction routed ordered navigation through the LMDB cursor.</p>

<h2 id="machine-identity"><a class="no-underline" href="#machine-identity">Machine identity</a></h2>
<p>The historical runs did not record manufacturer, model, CPU, memory, or OS.
Their machine type is therefore <code>UNRECORDED</code>. A current workstation profile
identifies an Alienware m16 R2, but it is a future-run input only and must not
be assigned retroactively to the historical measurements.</p>
<p>Future runs capture a machine profile beside the raw build and measurement
transcripts. Comparable rows require the machine label, engine state, dataset
size, operation, timer source, and evidence paths.</p>
<h2 id="publication-boundary"><a class="no-underline" href="#publication-boundary">Publication boundary</a></h2>
<p>This public summary is a reviewed projection of the development benchmark
record. The plan, eight-row ledger, machine profile, validator, runners, and
closeouts are preserved in
<a href="https://github.com/deraldg/x64base/tree/853937f1f612c1a41a8fbda2c8d99e55fe44b9a3">public source commit <code>853937f1</code></a>.
That source promotion does not prove that the navigation correction is part of
the current public runtime or a tagged release.</p>
