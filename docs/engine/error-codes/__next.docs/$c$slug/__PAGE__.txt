<h2 id="code-identity-hresult-style"><a class="no-underline" href="#code-identity-hresult-style">Code identity (HRESULT-style)</a></h2>
<p>Every xBase_64 error code is a packed 32-bit value with a stable ABI:</p>
<pre><code> 31      sign bit (always 1 for xBase_64 codes)
 30..29  severity   (2 bits)
 28..16  facility   (13 bits)
 15..0   code number (16 bits)
</code></pre>
<p>A code is written compactly as <strong><code>facility/0xNNNN</code></strong> — for example, the I/O
write failure is <code>io/0x0001</code>. Helper accessors (<code>get_severity</code>,
<code>get_facility</code>, <code>get_number</code>, <code>ok</code>, <code>failed</code>) decode the packed value; <code>ok()</code>
is severity <code>success</code>.</p>

<h2 id="how-errors-surface"><a class="no-underline" href="#how-errors-surface">How errors surface</a></h2>
<p>Commands raise a canonical code and a localized message through the <code>cmdout</code>
channel (<code>src/cli/command_output.hpp</code>): <code>emit_error(cmd, code, ...)</code> /
<code>emit_warning(...)</code> print the localized text and record the code via
<code>set_last_error</code>, so the same failure is inspectable through the <code>ERROR_STATUS</code>
command and (as it is wired) haltable under <code>SET ERRORSTOP</code>. Message text is
localized — see <a href="/docs/engine/messaging-and-localization">Messaging &#x26; Localization</a>.</p>
