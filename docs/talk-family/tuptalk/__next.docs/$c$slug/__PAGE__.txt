6:T47e,<h2 id="what-it-does"><a class="no-underline" href="#what-it-does">What it does</a></h2>
<p>TupTalk names the tuple-facing inspection workflows in the current system.</p>
<p>Its subject is not "tables in general." Its subject is the current row as a
readable tuple, projected record, or relation-derived row set. That includes:</p>
<ul>
<li>current-record tuple display</li>
<li>browser rows treated as tuples</li>
<li>exported tuple-shaped output</li>
<li>tuple validation and tuple-contract checks</li>
<li>relation-aware tuple output through <code>ERSATZ</code></li>
</ul>
<p>Real local command surfaces include:</p>
<pre><code class="language-text">SIMPLEBROWSE
SIMPLEBROWSE PAGE 20
SIMPLEBROWSE FOR GPA >= 3.0
SIMPLEBROWSE START KEY SMITH
SB
TUPLE
TUPTALK
TUPEXPORT
TUPVALIDATE
</code></pre>
<p>Use these pages to describe tuple inspection, row projection, export, and
relation-aware output. Do not describe TupTalk as a finished standalone CLI
unless that packaging exists.</p>
