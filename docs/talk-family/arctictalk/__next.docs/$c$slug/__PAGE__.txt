4:T59f,<h2 id="what-it-does"><a class="no-underline" href="#what-it-does">What it does</a></h2>
<p>ArcticTalk is the public umbrella for the DotTalk++ front-end family.</p>
<p>Within that umbrella, fox-derived subsystem names such as <code>Foxtalk</code> remain
valid lineage names, and repeatable maintenance/automation workflows remain part
of the same family.</p>
<p>Its subject is the workbench/front-end layer around the runtime:</p>
<ul>
<li>GUI and TUI presentation</li>
<li>shell-bridge and command-lane work</li>
<li>repeatable smoke, indexing, and maintenance workflows</li>
<li>helper tooling that makes the runtime easier to inspect, teach, and operate</li>
</ul>
<p>Real local command surfaces and workflow anchors include:</p>
<pre><code class="language-text">BUILDLMDB
BUILDLMDB CLEAN YES
BUILDLMDB FORCE YES
REINDEX
CMDHELPCHK
CMDHELPCHK REF
WORKSPACE OPEN DBF
WORKSPACE OPEN DBF CDX
WORKSPACE SAVE &#x3C;name>
WORKSPACE LOAD &#x3C;name>
</code></pre>
<p>The local source also includes DotScript scripts, smoke checks, maintenance
lanes, and generated proposal packages. Treat this as active workflow
infrastructure, not as a separate finished batch product.</p>
