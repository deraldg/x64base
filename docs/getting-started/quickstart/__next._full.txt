17:Tf4e,<p>The project is command-shell first. Use DotTalk++ scripts and commands rather than invented package-style examples.</p>
<p>This page separates public clone/build examples from Derald's local development paths. Local evidence pages may still cite <code>implementation checkout</code> because that is the active development tree, but a new reader should be able to start from a normal checkout.</p>
<h2 id="clone"><a class="no-underline" href="#clone">Clone</a></h2>
<pre><code class="language-powershell">git clone https://github.com/deraldg/x64base.git
cd x64base
</code></pre>
<p>The staging/publishing workflow currently mirrors active development into <code>local staging mirror</code>, but outside contributors should treat the GitHub checkout as the portable starting point.</p>
<h2 id="build-shape"><a class="no-underline" href="#build-shape">Build shape</a></h2>
<p>The CMake project is <code>DotTalkpp</code> and uses C++20.</p>
<p>Windows / Visual Studio build shape:</p>
<pre><code class="language-powershell">cmake -S . -B build -G "Visual Studio 17 2022" -A x64 -D DOTTALK_WITH_INDEX=ON -D DOTTALK_WITH_RELATIONS=ON -D BUILD_TESTING=ON
cmake --build build --config Release --target dottalkpp
</code></pre>
<p>WSL / Ubuntu build shape:</p>
<pre><code class="language-bash">sudo apt update
sudo apt install -y build-essential cmake ninja-build git pkg-config
cmake -S . -B build-posix -G Ninja \
  -D DOTTALK_WITH_INDEX=ON \
  -D DOTTALK_WITH_RELATIONS=ON \
  -D BUILD_TESTING=ON
cmake --build build-posix
</code></pre>
<p>macOS build shape:</p>
<pre><code class="language-bash">xcode-select --install
brew install cmake ninja pkg-config
cmake -S . -B build-macos -G Ninja \
  -D DOTTALK_WITH_INDEX=ON \
  -D DOTTALK_WITH_RELATIONS=ON \
  -D BUILD_TESTING=ON
cmake --build build-macos
</code></pre>
<p>GUI lanes are optional:</p>
<pre><code class="language-powershell">cmake -S . -B build-wx-check -G "Visual Studio 17 2022" -A x64 -D DOTTALK_WITH_GUI=ON -D DOTTALK_WITH_WX=ON -D DOTTALK_WITH_INDEX=ON
cmake --build build-wx-check --config Release
</code></pre>
<h2 id="run-a-script-example"><a class="no-underline" href="#run-a-script-example">Run a script example</a></h2>
<p>The repository contains many <code>.dts</code> examples and canaries. Good starting classes are:</p>
<pre><code class="language-text">dottalkpp\data\scripts\canaries\dottalk_beta0_smoke_baseline_0_1.dts
dottalkpp\data\scripts\canaries\workspace_load_x64_canary.dts
dottalkpp\data\scripts\canaries\smartlist_order_matrix.dts
tests\manual\dotscript\core\MCC_TIER1_CORE_SURFACE_AND_INDEX_SAFE_V1.DTS
tests\manual\dotscript\core\MCC_TIER2_OBSERVATION_CANARIES_V1.DTS
tests\manual\dotscript\core\MCC_TIER3_SANDBOX_MUTATION_CANARIES_V1.DTS
</code></pre>
<p>Run commands vary by current build output and launcher lane. The important public standard is that examples should name their required dataset, expected mutation level, and proof status.</p>
