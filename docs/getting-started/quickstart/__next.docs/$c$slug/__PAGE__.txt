6:Td38,<h2 id="clone-and-build"><a class="no-underline" href="#clone-and-build">Clone and build</a></h2>
<p>The fullest working build on the public <code>main</code> branch is <code>pro-md</code> — the core
engine plus LMDB indexing and the Turbo Vision UI. Use it if you want a working
database runtime.</p>
<pre><code class="language-powershell">git clone https://github.com/deraldg/x64base.git
cd x64base
cmake --preset pro-md
cmake --build --preset pro-md-Release --target dottalkpp
</code></pre>
<p>Configure-preset and build-preset names differ: <code>pro-md</code> configures,
<code>pro-md-Release</code> builds. For the smallest build instead, use <code>windows-core</code>
(or <code>core-vcpkg</code> on Linux, WSL, or macOS) — see
<a href="/docs/getting-started/installation">Installation</a>.</p>
<h2 id="build-the-sample-databases"><a class="no-underline" href="#build-the-sample-databases">Build the sample databases</a></h2>
<p>A fresh clone has no sample data until you build it. One script does the whole
job and is self-contained — it resets, extracts the canonical archive, and
builds all three DBF lanes (MS-DOS, VFP, x64) plus the x64 LMDB indexes:</p>
<pre><code class="language-powershell">.\dottalkpp\scripts\mcc\build_mcc_demo_bases.ps1
</code></pre>
<p>It warns before overwriting and waits for you to type <code>YES</code>.</p>
<h2 id="run-and-query"><a class="no-underline" href="#run-and-query">Run and query</a></h2>
<pre><code class="language-powershell">.\datarun.ps1
</code></pre>
<p><code>datarun.ps1</code> stages the freshly built <code>dottalkpp.exe</code> and its runtime DLLs into
<code>dottalkpp\bin</code>, then starts the shell. Set the lane and query:</p>
<pre><code class="language-text">DO X64                  &#x26;&#x26; sets the x64 lane (dbf, indexes, lmdb)
USE STUDENTS
SET INDEX TO STUDENTS
SET ORDER TO TAG LNAME
SMARTLIST 10
</code></pre>
<p>Run <code>DO X64</code> first: the default DBF path is <code>data\dbf</code>, not a lane, so a bare
<code>USE STUDENTS</code> will not find the table. <code>DO X64</code> points the DBF, index, and LMDB
paths at the x64 lane (<code>DO X32</code> and <code>DO VFP</code> select the other flavors). Ordered
by <code>LNAME</code>, record one reads <code>Anderson</code>.</p>
<p>Use <code>HELP</code> followed by a command name for the current command contract. Mutation
commands such as <code>REPLACE</code>, <code>CALCWRITE</code>, <code>DELETE</code>, <code>COMMIT</code>, and <code>PACK</code> should be
exercised only against disposable data — never the sample foundation.</p>
<h2 id="reality-check"><a class="no-underline" href="#reality-check">Reality check</a></h2>
<ul>
<li><code>main</code> is the canonical public source.</li>
<li>A green CI run proves only the selected build/test profiles.</li>
<li>On x64, ordered reads are LMDB-backed. The shipped <code>.cnx</code> / <code>.cdx</code> containers
let you open and read a table, but <code>SET ORDER TO TAG</code> on x64 needs the LMDB
environments the databuild above generates (they are derived, not shipped).</li>
<li><code>COMMIT</code> is a best-effort buffer apply operation, not an atomic transaction
across DBF, memo, and index stores.</li>
</ul>
