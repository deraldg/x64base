17:Tdf5,<p>The authoritative local source tree is <code>D:\code\ccode</code>.</p>
<p>The project is a CMake C++20 project named <code>DotTalkpp</code>. The root CMake file exposes build options for indexing, relations, education, Turbo Vision, GUI-neutral services, wxWidgets, tests, and optional Python bindings.</p>
<h2 id="requirements"><a class="no-underline" href="#requirements">Requirements</a></h2>
<ul>
<li>64-bit Windows development environment.</li>
<li>CMake 3.21 or newer.</li>
<li>Visual Studio 2022 or another configured C++20 toolchain.</li>
<li>POSIX-compatible shell and build tools for WSL, Ubuntu, Linux, and macOS builds.</li>
<li>Optional vcpkg packages for local dependencies.</li>
<li>Optional Turbo Vision source/package if building TUI surfaces.</li>
<li>Optional wxWidgets when building the wx frontend.</li>
</ul>
<h2 id="cli-build-shape"><a class="no-underline" href="#cli-build-shape">CLI build shape</a></h2>
<pre><code class="language-powershell">cd D:\code\ccode
cmake -S . -B build -G "Visual Studio 17 2022" -A x64 -D DOTTALK_WITH_INDEX=ON -D DOTTALK_WITH_RELATIONS=ON -D BUILD_TESTING=ON
cmake --build build --config Release --target dottalkpp
</code></pre>
<h2 id="gui-build-shape"><a class="no-underline" href="#gui-build-shape">GUI build shape</a></h2>
<pre><code class="language-powershell">cd D:\code\ccode
cmake -S . -B build-wx-check -G "Visual Studio 17 2022" -A x64 -D DOTTALK_WITH_GUI=ON -D DOTTALK_WITH_WX=ON -D DOTTALK_WITH_INDEX=ON
cmake --build build-wx-check --config Release
</code></pre>
<h2 id="wsl--ubuntu-posix-build-requirements"><a class="no-underline" href="#wsl--ubuntu-posix-build-requirements">WSL / Ubuntu POSIX build requirements</a></h2>
<p>Use WSL2 or a native Ubuntu environment when testing the POSIX build lane.</p>
<p>Expected packages:</p>
<pre><code class="language-bash">sudo apt update
sudo apt install -y build-essential cmake ninja-build git pkg-config
</code></pre>
<p>Optional packages depend on the feature lane being tested:</p>
<pre><code class="language-bash">sudo apt install -y liblmdb-dev libwxgtk3.2-dev python3-dev python3-venv
</code></pre>
<p>Build shape:</p>
<pre><code class="language-bash">cd /mnt/d/code/ccode
cmake -S . -B build-posix -G Ninja \
  -D DOTTALK_WITH_INDEX=ON \
  -D DOTTALK_WITH_RELATIONS=ON \
  -D BUILD_TESTING=ON
cmake --build build-posix
</code></pre>
<p>The POSIX lane should keep path handling, case sensitivity, newline behavior, file locking, and terminal behavior visible in test notes.</p>
<h2 id="macos-build-requirements"><a class="no-underline" href="#macos-build-requirements">macOS build requirements</a></h2>
<p>Use Xcode Command Line Tools plus Homebrew packages for the macOS lane.</p>
<pre><code class="language-bash">xcode-select --install
brew install cmake ninja pkg-config
</code></pre>
<p>Optional feature packages:</p>
<pre><code class="language-bash">brew install lmdb wxwidgets python
</code></pre>
<p>Build shape:</p>
<pre><code class="language-bash">cd ~/code/ccode
cmake -S . -B build-macos -G Ninja \
  -D DOTTALK_WITH_INDEX=ON \
  -D DOTTALK_WITH_RELATIONS=ON \
  -D BUILD_TESTING=ON
cmake --build build-macos
</code></pre>
<p>macOS notes should track compiler version, SDK version, filesystem case behavior, terminal behavior, and optional GUI backend status.</p>
