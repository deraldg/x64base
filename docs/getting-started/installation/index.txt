16:T146c,<p>The public <a href="https://github.com/deraldg/x64base"><code>main</code> branch</a> is the portable
starting point. The project requires CMake 3.21 or newer and a C++20 compiler.</p>
<h2 id="requirements"><a class="no-underline" href="#requirements">Requirements</a></h2>
<ul>
<li>
<p>Git and CMake 3.21 or newer.</p>
</li>
<li>
<p>Visual Studio 2022 on Windows, or GCC/Clang plus Ninja on Linux, WSL, and macOS.</p>
</li>
<li>
<p>vcpkg for the core JSON/SQLite dependencies and optional feature lanes.</p>
</li>
<li>
<p>The vcpkg environment set. The presets reference two variable names for
historical reasons; set <strong>both</strong> to your vcpkg checkout so every preset
resolves:</p>
<pre><code class="language-powershell">$env:VCPKG_ROOT              = "path\to\vcpkg"
$env:VCPKG_INSTALLATION_ROOT = "path\to\vcpkg"
</code></pre>
</li>
</ul>
<p>Dependencies are declared in <code>vcpkg.json</code> and restored on the first configure
(manifest mode).</p>
<h2 id="full-working-build-windows"><a class="no-underline" href="#full-working-build-windows">Full working build (Windows)</a></h2>
<p><code>pro-md</code> is the fullest build on <code>main</code> — the core engine plus LMDB indexing and
the Turbo Vision UI. Use it if you want a working database runtime.</p>
<pre><code class="language-powershell">git clone https://github.com/deraldg/x64base.git
cd x64base
cmake --preset pro-md
cmake --build --preset pro-md-Release --target dottalkpp
</code></pre>
<p>Configure-preset and build-preset names differ — the build preset carries a
<code>-Release</code> suffix (<code>pro-md</code> configures, <code>pro-md-Release</code> builds).</p>
<h2 id="windows-core"><a class="no-underline" href="#windows-core">Windows core</a></h2>
<p>The smallest public build proof — no LMDB index commands, no Turbo Vision,
wxWidgets, Python bindings, or developer-only command visibility:</p>
<pre><code class="language-powershell">git clone https://github.com/deraldg/x64base.git
cd x64base
cmake --preset windows-core
cmake --build --preset windows-core --parallel
ctest --preset windows-core
</code></pre>
<p>LMDB itself remains a base dependency because the runtime message catalog
maintains LMDB mirrors.</p>
<h2 id="linux-wsl-or-macos-core"><a class="no-underline" href="#linux-wsl-or-macos-core">Linux, WSL, or macOS core</a></h2>
<pre><code class="language-bash">git clone https://github.com/deraldg/x64base.git
cd x64base
cmake --preset core-vcpkg
cmake --build --preset core-vcpkg --parallel
ctest --preset core-vcpkg
</code></pre>
<h2 id="optional-indexing-lane"><a class="no-underline" href="#optional-indexing-lane">Optional indexing lane</a></h2>
<pre><code class="language-bash">cmake --preset index-vcpkg
cmake --build --preset index-vcpkg --parallel
ctest --preset index-vcpkg
</code></pre>
<p>The vcpkg manifest exposes separate <code>index</code>, <code>tv</code>, <code>wx</code>, and <code>python</code> features.
Do not place personal dependency paths in tracked build files; use environment
variables or an untracked <code>CMakeUserPresets.json</code>.</p>
<h2 id="after-the-build"><a class="no-underline" href="#after-the-build">After the build</a></h2>
<p>A fresh clone has no sample data until you build it. From the repo root, run the
self-contained databuild, then start the runtime — see the
<a href="/docs/getting-started/quickstart">Quickstart</a> for the full sequence:</p>
<pre><code class="language-powershell">.\dottalkpp\scripts\mcc\build_mcc_demo_bases.ps1
.\datarun.ps1
</code></pre>
<p>Inside the shell, run <code>DO X64</code> to select the x64 lane before <code>USE STUDENTS</code>: the
default DBF path is <code>data\dbf</code>, not a lane, so a bare <code>USE</code> will not find the
table.</p>
<h2 id="editions-in-development--not-yet-in-this-repository"><a class="no-underline" href="#editions-in-development--not-yet-in-this-repository">Editions (in development — not yet in this repository)</a></h2>
<p>A larger build-composition system is being developed: named <strong>products</strong>
(<code>LEAN</code>, <code>PROFESSIONAL</code>, <code>EDUCATIONAL</code>, <code>DEVELOPMENT</code>) and explicit <strong>index
modes</strong> (<code>NONE</code>, <code>LEGACY</code>, <code>LMDB</code>), selected through dedicated <code>windows-lean-*</code>,
<code>windows-educational-lmdb</code>, and <code>windows-development-lmdb</code> presets and the
<code>DOTTALK_PRODUCT</code> / <code>DOTTALK_INDEX_MODE</code> options. <code>LEAN</code> strips most education,
maintenance, and developer surfaces down to a small distributable runtime.</p>
<p>This system is <strong>not on the public <code>main</code> branch yet</strong> — a fresh clone of <code>main</code>
will not have <code>DOTTALK_PRODUCT</code> or the <code>windows-lean-*</code> presets, so build only
the presets listed above for now. Once the edition system is published and
certified with a cold-clone build, this page will document the full
product × index-mode build commands.</p>
