7:T100b,<p>DotTalk++ / x64base is a working educational and research runtime for xBase-style database ideas. It is an object-oriented hybrid: areas, workspaces, memos, indexes, relations, metadata, and GUI/script surfaces are runtime objects wrapped around DBF-family interoperability.</p>
<p>It is not currently positioned as a finished commercial database product. It is an active beta system that exposes database layers directly: table files, records, fields, work areas, indexes, relations, metadata, HELP, validation, scripts, browsers, and simple UI surfaces.</p>
<p>The central thesis is that x64base is a database system that uses its own
metadata and documentation infrastructure to describe, validate, and
increasingly prove itself. The 64-bit xBase lineage matters, but the larger
project is the feedback loop between runtime behavior, metadata, documentation,
proof, and teaching.</p>
<p>The system is being built in co-development with its documentation. SelfDoc and the Master Documentation Organizer are part of the working system: the database engine develops and documents the same structures it is building. The goal is an open, cross-platform architecture that can be studied, extended, validated, and rebuilt across Windows, WSL/Linux, Ubuntu, and macOS.</p>
<h2 id="what-the-local-project-contains"><a class="no-underline" href="#what-the-local-project-contains">What the local project contains</a></h2>
<ul>
<li><strong>DotTalk++ command shell</strong> - the canonical runtime surface for commands, scripts, HELP, browsing, validation, and workspace state.</li>
<li><strong>DotScript</strong> - the repeatable script-language product for command files, <code>IF</code>, loops, variables, line continuation, comments, scans, one-level subscript nesting, CSV/DBF import-export, and automation.</li>
<li><strong>x64base / DBF runtime</strong> - DBF-style table handling across classic/MS-DOS DBF, VFP DBF, and x64 DBF_64 work, cursor movement, record mutation, work areas, and table inspection.</li>
<li><strong>xindex</strong> - INX, CNX, CDX, and LMDB-backed indexing work.</li>
<li><strong>Workspaces and areas</strong> - workspace wrappers over active <code>DbArea</code>-style objects, including table state, cursor state, indexes, relations, memos, and metadata.</li>
<li><strong>Memo subsystem</strong> - object-oriented memo architecture with active canary coverage.</li>
<li><strong>Extension hooks</strong> - student code hooks plus custom field type hooks for validators, display behavior, import/export semantics, and documentation metadata.</li>
<li><strong>Metadata and self-documentation</strong> - command reflection, <code>CMDHELPCHK</code>, SYS* metadata work, data dictionary readers, and congruence validation.</li>
<li><strong>Table listing and browser surfaces</strong> - <code>SMARTLIST</code>, <code>SMARTBROWSE</code>, <code>SM</code>, <code>SMART</code>, <code>SIMPLEBROWSE</code>, <code>SB</code>, <code>BROWSE</code>, <code>BROWSER</code>, <code>BROWSETUI</code>, <code>ERSATZ</code>, and <code>RBROWSE</code>.</li>
<li><strong>UI lanes</strong> - Turbo Vision-style TUI, wxWidgets workbench work, and Python/Tk preview tools.</li>
<li><strong>Python integration</strong> - <code>pydottalk</code> binding experiments and smoke scripts.</li>
</ul>
