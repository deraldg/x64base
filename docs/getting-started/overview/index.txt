a:T993,<p>DotTalk++ / x64base is a working educational and research runtime for xBase-style database ideas.</p>
<p>It is not currently positioned as a finished commercial database product. It is an active beta system that exposes database layers directly: table files, records, fields, work areas, indexes, relations, metadata, HELP, validation, scripts, browsers, and simple UI surfaces.</p>
<h2 id="what-the-local-project-contains"><a class="no-underline" href="#what-the-local-project-contains">What the local project contains</a></h2>
<ul>
<li><strong>DotTalk++ command shell</strong> - the canonical runtime surface for commands, scripts, HELP, browsing, validation, and workspace state.</li>
<li><strong>x64base / DBF runtime</strong> - DBF-style table handling, x32 and x64-family work, cursor movement, record mutation, work areas, and table inspection.</li>
<li><strong>xindex</strong> - INX, CNX, CDX, and LMDB-backed indexing work.</li>
<li><strong>Memo subsystem</strong> - object-oriented memo architecture with active canary coverage.</li>
<li><strong>Metadata and self-documentation</strong> - command reflection, <code>CMDHELPCHK</code>, SYS* metadata work, data dictionary readers, and congruence validation.</li>
<li><strong>Browser surfaces</strong> - <code>ERSATZ</code>, <code>RBROWSE</code>, <code>SMARTBROWSE</code>, <code>ARTBROWSE</code>, <code>SIMPLEBROWSE</code>, and <code>SB</code>.</li>
<li><strong>UI lanes</strong> - Turbo Vision-style TUI, wxWidgets workbench work, and Python/Tk preview tools.</li>
<li><strong>Python integration</strong> - <code>pydottalk</code> binding experiments and smoke scripts.</li>
</ul>
