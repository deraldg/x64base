<p>The candidate provider is <strong>Moodle LMS</strong>, but this is not a claim that LabTalk
currently connects to Moodle. No Moodle site URL, access token, student record,
grade, or live network operation is stored or enabled in this lane.</p>
<h2 id="communications-boundary"><a class="no-underline" href="#communications-boundary">Communications Boundary</a></h2>
<pre><code class="language-text">Laboratory Campus lesson or assignment intent
                    |
                    v
          labtalk.lms.message.v1
                    |
                    v
             local JSON outbox
                    |
                    v
           reserved transport -X-> Moodle external services
                                  future and separately approved
</code></pre>
<p>A queued file records local integration intent. It is not proof that Moodle or
any other provider accepted an operation. A future adapter must return a
provider receipt before the campus can report successful delivery.</p>
<h2 id="local-interfaces"><a class="no-underline" href="#local-interfaces">Local Interfaces</a></h2>
<p>From the repository's <code>labtalk</code> directory, the pipe can be inspected without
network delivery:</p>
<pre><code class="language-powershell">python ..\src\labtalk\lms\cli.py --queue .\lms\queue\outbox status
python ..\src\labtalk\lms\cli.py --queue .\lms\queue\outbox list
</code></pre>
<p>The portal exposes the same status boundary:</p>
<pre><code class="language-powershell">python .\portal\labtalk_portal.py --run-item lms.pipe.status
</code></pre>
<p>The CLI also supports controlled enqueueing of development envelopes. Enqueueing
does not activate a provider transport.</p>

<h2 id="gates-before-live-delivery"><a class="no-underline" href="#gates-before-live-delivery">Gates Before Live Delivery</a></h2>
<ol>
<li>Select and administer a sandbox Moodle site.</li>
<li>Approve the exact Moodle external functions LabTalk may invoke.</li>
<li>Map each LabTalk operation to one approved provider function.</li>
<li>Keep tokens outside source control, queue payloads, transcripts, and logs.</li>
<li>Add idempotency, retry, receipt, redaction, and audit behavior.</li>
<li>Prove the adapter in a sandbox without student or grade data.</li>
<li>Perform a separate privacy, security, and instructional review before any
live-delivery flag can change.</li>
</ol>
