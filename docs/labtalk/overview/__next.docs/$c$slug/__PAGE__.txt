e:Te4a,<p>The <strong>Laboratory Campus</strong> is the learning-lab and collaboration framing for the
x64base / DotTalk++ ecosystem. LabTalk® remains the project mark and legacy name,
but the campus is the public frame to emphasize.</p>
<p>It is not just a single app. It is an alpha hybrid of planning material,
implemented pieces, in-development labs, datasets, runtime demonstrations, help
systems, GUI experiments, and proof artifacts that make computing systems easier
to study and easier to collaborate on.</p>
<p><img src="/images/labtalk/labtalk_campus_map_v0.svg" alt="LabTalk Laboratory Campus"></p>
<h2 id="laboratory-campus"><a class="no-underline" href="#laboratory-campus">Laboratory Campus</a></h2>
<p>Think of it as a campus:</p>
<ul>
<li><strong>Runtime Systems Lab</strong> - DotTalk++ commands, scripts, work areas, tables, records, indexes, and relations.</li>
<li><strong>Historical Data Systems Lab</strong> - punch cards, COBOL, CODASYL, xBase, SQL, ERP, cloud computing, and AI-era data literacy.</li>
<li><strong>Self-Documenting Systems Lab</strong> - comments, contracts, HELP, CMDHELP, CMDHELPCHK, metadata, and proof readback.</li>
<li><strong>Dataset Library</strong> - small inspectable datasets for lessons and repeatable labs.</li>
<li><strong>Case Library</strong> - engineering and historical cases that connect live demonstrations to real system stories.</li>
<li><strong>GUI and Portal Lab</strong> - local launchers, dashboards, and front-end experiments that expose the same underlying evidence.</li>
</ul>
<p>The first public history path is the <a href="/docs/labtalk/database-evolution">Database Evolution Path</a>,
which integrates the LabTalk / DotTalk++ storyboard deck as a teaching artifact
from punch cards and COBOL through xBase, enterprise data, cloud computing, and AI.</p>
<p>The <a href="/docs/labtalk/education-features">Education Features</a> page now exposes
source-defined teaching surfaces such as <code>IDX</code> timed index/sort labs, student
command/function hooks, <code>SORT</code> lab behavior, and pre/post polling hooks.</p>
<p>The <a href="/docs/labtalk/runtime-evidence">Runtime Evidence Gallery</a> records reviewed screenshots from Arctic, the local campus portal, workspace loading, table browsing, tuple output, and GUI/TUI readback for use by the manual and website.</p>
<p><img src="/images/sdlc/laboratory-campus-evolution-spine.svg" alt="Laboratory Campus evolution spine"></p>
<h2 id="core-pattern"><a class="no-underline" href="#core-pattern">Core Pattern</a></h2>
<p>Campus material should follow a proof-aware path:</p>
<p><img src="/images/labtalk/labtalk_proof_path_v0.svg" alt="LabTalk proof-aware learning path"></p>
<pre><code class="language-text">concept -> app -> dataset -> command -> proof -> case -> lesson
</code></pre>
<p>This keeps lessons connected to live or reviewed evidence instead of becoming
detached prose.</p>
<h2 id="current-status"><a class="no-underline" href="#current-status">Current Status</a></h2>
<p>The Laboratory Campus is an active alpha direction. Public x64base pages should
describe it as a campus model, collaboration area, and learning-lab program, not
as a finished packaged distribution unless that packaging has been explicitly
released.</p>
