a:T6e5,<p>The SelfDoc lane records how DotTalk++ turns working knowledge into durable,
proof-backed documentation. It connects chat decisions, source usage blocks,
<code>@dottalk.contract</code> annotations, HELP/CMDHELP evidence, and metadata import
candidates to contract registration, proof promotion, generated reports, and
LabTalk publication.</p>

<h2 id="current-flow"><a class="no-underline" href="#current-flow">Current Flow</a></h2>
<pre><code class="language-text">Intake -> Register -> Prove -> Drift Check
SelfDoc -> metacollect -> metadata_collection_v0 -> reports
Reports -> Draw.io / Mermaid -> LabTalk -> x64base.com
</code></pre>
<h2 id="evidence-rules"><a class="no-underline" href="#evidence-rules">Evidence Rules</a></h2>
<p>The lane keeps two guardrails visible:</p>
<ul>
<li>report-only SelfDoc pipelines must not mutate DBFs, rebuild HELP, change
CMDHELPCHK, repair source, or promote live metadata without review;</li>
<li>documentation must not claim runtime proof unless a test, command, or
reproducible report proves it.</li>
</ul>
