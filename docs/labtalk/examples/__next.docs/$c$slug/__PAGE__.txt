<h2 id="database-literacy-starter"><a class="no-underline" href="#database-literacy-starter">Database Literacy Starter</a></h2>
<p>Purpose:</p>
<pre><code class="language-text">Teach tables, records, fields, order, search, scan, and output.
</code></pre>
<p>Campus path:</p>
<pre><code class="language-text">EDREF MODEL
-> EDREF TABLE_RECORD_FIELD
-> EDREF INDEX
-> EDREF SCAN
-> DotTalk++ runtime transcript
-> student-facing lesson
</code></pre>
<p>The first local LabTalk portal slice already uses this path to run a read-only
DotTalk++ script and capture a proof transcript.</p>
<h2 id="comments-to-contracts-to-help"><a class="no-underline" href="#comments-to-contracts-to-help">Comments to Contracts to HELP</a></h2>
<p>Purpose:</p>
<pre><code class="language-text">Show how source comments become structured contracts, HELP output, and validation evidence.
</code></pre>
<p>Campus path:</p>
<pre><code class="language-text">source comment
-> @dottalk.usage v1
-> comments evidence
-> CMDHELP artifact
-> CMDHELPCHK validation
-> LabTalk lesson
</code></pre>
<p>This is the Self-Documenting Systems Lab.</p>
<h2 id="historical-data-systems-trail"><a class="no-underline" href="#historical-data-systems-trail">Historical Data Systems Trail</a></h2>
<p>Purpose:</p>
<pre><code class="language-text">Connect database history to live or simulated system behavior.
</code></pre>
<p>Campus path:</p>
<pre><code class="language-text">Punch cards / fixed records
-> COBOL / business reports
-> CODASYL / network traversal
-> xBase / DBF records and indexes
-> SQL / ERP / industrial data
-> cloud computing / distributed services
-> AI-era explainability
</code></pre>
<p>The <a href="/docs/labtalk/database-evolution">Database Evolution Path</a> is the first
public version of this trail. It uses the LabTalk / DotTalk++ storyboard deck to
connect punch cards, COBOL, ARPANET, JUMPS, CODASYL, xBase, ERP, cloud computing,
and AI literacy.</p>
