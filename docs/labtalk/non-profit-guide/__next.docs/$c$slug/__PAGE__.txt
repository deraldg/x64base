6:T907,<p>The Laboratory Campus is intended to support non-profit, educational, and
community learning use without blurring the line between learning material and
core engine behavior. LabTalk remains the license/project mark for this area.</p>
<h2 id="suggested-onboarding"><a class="no-underline" href="#suggested-onboarding">Suggested Onboarding</a></h2>
<ol>
<li>Start with <a href="/docs/getting-started/overview">Getting Started</a>.</li>
<li>Read the <a href="/docs/labtalk/overview">Laboratory Campus Overview</a>.</li>
<li>Use <a href="/docs/dottalk/language-guide">DotTalk++</a> for command and script literacy.</li>
<li>Use the Talk-family pages to learn tuples, tables, relations, and display concepts.</li>
<li>Work through one small lab before expanding to larger datasets.</li>
<li>Review the concise <a href="/licensing">project notice</a>.</li>
</ol>
<h2 id="best-practices"><a class="no-underline" href="#best-practices">Best Practices</a></h2>
<ul>
<li>Keep sample datasets small and inspectable.</li>
<li>Use scripts to make lessons repeatable.</li>
<li>Label historical simulations as simulations.</li>
<li>Keep proof transcripts separate from lesson prose.</li>
<li>Do not treat curriculum incubator material as runtime truth until it is reviewed.</li>
<li>Prefer read-only labs for first exposure.</li>
</ul>
