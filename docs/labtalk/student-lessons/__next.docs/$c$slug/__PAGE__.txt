<h2 id="first-lesson-shape"><a class="no-underline" href="#first-lesson-shape">First Lesson Shape</a></h2>
<p>Every promoted student lesson should answer:</p>
<ol>
<li>What concept is the student learning?</li>
<li>What app, lab, command, dataset, or artifact exposes it?</li>
<li>What should the student do?</li>
<li>What output or observation should they expect?</li>
<li>What proof backs the claim?</li>
<li>What safety or mutation warning applies?</li>
<li>What is still a draft, simulation, or review-needed claim?</li>
</ol>
<h2 id="candidate-path-records-fields-and-tables"><a class="no-underline" href="#candidate-path-records-fields-and-tables">Candidate Path: Records, Fields, and Tables</a></h2>
<p>The first beginner lesson follows the database literacy starter:</p>
<pre><code class="language-text">concept.database.table
-> app.dottalkpp.runtime
-> dataset.students
-> USE / FIELDS / STRUCT / LIST
-> proof.lab.database_literacy_starter.first_run
-> lesson.student.records_fields_tables
</code></pre>
<p>The student goal is simple: inspect a table, identify fields and records, then
explain the difference between table structure and current row state.</p>
<h2 id="candidate-path-self-documenting-systems"><a class="no-underline" href="#candidate-path-self-documenting-systems">Candidate Path: Self-Documenting Systems</a></h2>
<p>The first advanced lesson follows the SelfDoc lab:</p>
<pre><code class="language-text">source comment
-> @dottalk.usage contract
-> comments evidence
-> CMDHELP
-> CMDHELPCHK
-> proof.lab.selfdoc.comments_to_contracts.first_run
-> lesson.student.selfdoc_comments_to_help
</code></pre>
<p>The student goal is to classify a claim as source-defined, help-documented,
validated, or review-needed.</p>
<h2 id="review-needed"><a class="no-underline" href="#review-needed">Review Needed</a></h2>
<p>Useful student-lesson review includes:</p>
<ul>
<li>CS review for data structures, runtime state, algorithms, and systems framing.</li>
<li>database review for DBF records, indexing, relations, mutation safety, and lineage.</li>
<li>technical writing review for lesson structure, terminology, setup, and clarity.</li>
<li>general education review for accessibility, sequencing, assessment, and classroom fit.</li>
</ul>
