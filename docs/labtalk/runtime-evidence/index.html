Runtime Evidence Gallery

Screenshot evidence for DotTalk++ Workbench, Arctic TUI, LabTalk Portal, workspaces, table browsing, tuple output, and SelfDoc-facing manual/website proof.

This page records visual evidence from the local runtime. These images are evidence artifacts for the manual and website, not product screenshots claiming a finished release.

Evidence date: July 3, 2026.

Screenshot media is curated like the rest of the documentation system: stable filename, evidence ID, source capture, caption, interpretation boundary, and review-before-promotion. Duplicate captures are not repeated unless they show a distinct runtime state.

DotTalk++ Workbench Workspace Areas

Observed evidence:

  • DotTalk++ Workbench GUI evidence is running against TEACHERS.DBF. The captured build/window may still show the older Arctic code-name text; the product-family name is DotTalk++ Workbench.
  • Workspace pane lists 12 DBF areas: BUILDING.DBF, CLASSES.DBF, COURSES.DBF, DEPT.DBF, ENROLL.DBF, MAJORS.DBF, ROOMS.DBF, STUD_MAJ.DBF, STUDENTS.DBF, TASSIGN.DBF, TEACHERS.DBF, and TERMS.DBF.
  • Runtime output reports WORKSPACE LOAD: restored 12 area(s) and 15 relation(s).
  • GUI mirror reports 12 GUI areas created from the mcc_x64.dtschema workspace.

Table Browser and CDX/LMDB Evidence

Observed evidence:

  • Browse grid displays TEACHERS.DBF fields including TID, LNAME, FNAME, DEPT_ID, EMAIL, HIRE_DATE, and TENURED.
  • Status bar reports 20 rows shown of 20.
  • Status bar reports active ordered state: Order: ASC tag TID.
  • Output pane shows SET ORDER CDX metadata mismatch diagnostics with a .cdx container and .cdx.d LMDB environment path. This is useful evidence for the CDX/LMDB diagnostics lane and should be documented as a diagnostic state, not a clean success state.

Command Output Readback

Observed evidence:

  • The output pane preserves diagnostic text for later manual or SelfDoc review.
  • This supports the policy that GUI evidence should cite visible runtime text, not only visual layout.

Arctic TUI Tuple and Record Browser

Observed evidence:

  • Terminal/TUI shell shows the Arctic code-name surface with menus and command/output panes.

DotTalk++ CLI Startup and BibleTalk Evidence

Observed evidence:

  • DotTalk++ CLI starts with the dottalk++ beta-0 banner and DotTalk++ build timestamp.
  • Runtime reports SQLite availability and version 3.50.4.
  • EDU_BIBLETALK opens the SQLite seed biblebase\bible_kjv_x64.sqlite.
  • Startup/readback shows education-facing CLI output, including the ASCII display, BibleTalk quote, HELP, USE, SELECT <n>, AREA, COLOR, ABOUT, and QUIT prompt text.
  • Developer/status hints include CMDHELPCHK, GPS, WORKSPACE, and ERSATZ.

DotTalk++ CLI Workspace and Relation Load

Observed evidence:

  • DotTalk++ CLI runs ersatz load.
  • SETPATH readback shows x64 DBF, INDEXES, and LMDB roots.
  • Workspace close/reload output is visible before relation restoration.
  • REL: lines enumerate table relationships such as STUDENTS -> ENROLL ON SID, CLASSES -> TEACHERS ON TID, and ROOMS -> BUILDING ON BLDG.
  • Runtime reports WORKSPACE LOAD: restored 12 area(s) and 15 relation(s).
  • ERSATZ loads default.erz with SETUP=x64, WORKSPACE=mcc_x64.dtschema, and the path enroll -> classes -> tassign -> teachers.

DotTalk++ CLI Relation Tree and Tuple Grid

Observed evidence:

  • DotTalk++ CLI shows the current root record with fields including SID, LNAME, FNAME, DOB, GENDER, MAJOR, ENROLL_D, GPA, and EMAIL.
  • Relation tree output expands from STUDENTS through ENROLL, CLASSES, COURSES, DEPT, TERMS, ROOMS, BUILDING, TEACHERS, TASSIGN, STUD_MAJ, and MAJORS.
  • Descendant summary and tuple grid are visible in the CLI, separate from the TUI surface.
  • Tuple grid includes joined fields such as STUDENTS.LNAME, STUDENTS.FNAME, enroll.CLS_ID, GRADE, classes.CLS_ID, tassign.CLS_ID, TID, and teachers.LNAME.

  • DotTalk++ Workbench GUI surfaces.

  • Table browsing and SmartList/browser lanes.
  • CDX/LMDB order diagnostics.
  • LabTalk Portal and documentation index surfaces.
