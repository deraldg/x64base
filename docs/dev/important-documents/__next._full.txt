1e:T1986,<p>This page is the public documentation shelf for the most important document
layers behind x64base, DotTalk++, DotScript, DotTalk++ Workbench, and the
Laboratory Campus.</p>
<p>The website is not meant to be a hand-written brochure. It is a reviewed
publication surface built from the same vertical documentation system used by
the runtime: source comments, usage contracts, HELP, metadata, SelfDoc reports,
MDO organization, manualgen, diagrams, and AI-friendly summaries.</p>
<p><img src="/images/sdlc/parallel-lanes-documentation-metadata-sdlc.jpg" alt="Parallel lanes for documentation and metadata SDLC from definition through manuals and diagrams"></p>
<p>The most important documents are not one flat folder. They move through parallel
lanes: source/runtime proof, HELP explanation, metadata normalization, SelfDoc
review, provenance preservation, and promotion into manuals, diagrams, and the
website.</p>
<h2 id="document-layers"><a class="no-underline" href="#document-layers">Document Layers</a></h2>













































<table><thead><tr><th>Layer</th><th>Role</th><th>Website Status</th></tr></thead><tbody><tr><td>Source and runtime truth</td><td>C++ source, command registration, headers, contracts, runtime transcripts, and command behavior.</td><td>Referenced and summarized; not fully mirrored.</td></tr><tr><td>HELP and metadata</td><td><code>HELP</code>, <code>CMDHELP</code>, command/function catalogs, usage contracts, field/type metadata, and validation readback.</td><td>Command and function pages should be regenerated from this layer.</td></tr><tr><td>SelfDoc reports</td><td>Report-only extracts for features, contracts, comments, evidence, diagrams, and crosswalks.</td><td>Selected reports are promoted after review.</td></tr><tr><td>MDO organization</td><td>Master Documentation Organizer structure for lanes, manuals, evidence, sections, and promotion decisions.</td><td>Public pages should cite the organized lane, not raw scanner guesses.</td></tr><tr><td>AI-friendly artifacts</td><td>Compact summaries, manifests, crosswalks, inventories, and review notes designed for model-assisted maintenance.</td><td>Should be exposed when reviewed and clearly marked as derivative.</td></tr><tr><td>Manualgen outputs</td><td>Generated manual inventories, validation reports, dry-runs, and accepted manual sections.</td><td>Manuals are planned for publication as reviewed outputs.</td></tr><tr><td>Website derivatives</td><td>Public pages under <code>x64base.com</code> that present reviewed, status-labeled slices of the above.</td><td>This site is the current publication surface.</td></tr></tbody></table>
<h2 id="current-public-anchors"><a class="no-underline" href="#current-public-anchors">Current Public Anchors</a></h2>
<ul>
<li><a href="/docs/dev/project-truth">Current Project Truth</a> records conservative wording rules and the observed implementation state.</li>
<li><a href="/docs/dev/current-lanes">Current Work Lanes</a> lists active, canary, planned, and integration lanes without crowding the front page.</li>
<li><a href="/docs/dev/website-documentation-matrix">Website Documentation Matrix</a> maps website sections to source lanes, data-mining feeds, proof artifacts, and publication status.</li>
<li><a href="/docs/dev/selfdoc-feed-pipeline">SelfDoc Feed Pipeline</a> documents the repeatable report-only path from source to manuals and website pages.</li>
<li><a href="/docs/dev/selfdoc-website-publication">SelfDoc Website Publication</a> defines the boundary between implementation truth and public website copy.</li>
<li><a href="/docs/dev/coding-standards">Coding Standards</a> records contracts, usage comments, header conventions, and safeguards.</li>
<li><a href="/docs/labtalk/runtime-evidence">Runtime Evidence Gallery</a> curates screenshots as evidence artifacts, not marketing screenshots.</li>
<li><a href="/docs/engine/feature-crosswalk">Engine Feature Crosswalk</a> summarizes engine features from a generated SelfDoc report.</li>
<li><a href="/docs/dottalk/command-catalog">Command Catalog</a> and <a href="/docs/dottalk/function-catalog">Function Catalog</a> are public catalog derivatives that should eventually be regenerated from HELP/metadata/contract evidence.</li>
</ul>
<h2 id="manual-publication-plan"><a class="no-underline" href="#manual-publication-plan">Manual Publication Plan</a></h2>
<p>Manuals should appear here only after they are generated, validated, and
reviewed. The intended path is:</p>
<pre><code class="language-text">source behavior
-> comments and @dottalk.usage / @dottalk.contract headers
-> HELP, CMDHELP, metadata, and comments evidence
-> CMDHELPCHK / SelfDoc validation reports
-> MDO section organization
-> manualgen inventory / validate / dry-run
-> reviewed manual section
-> website manual page or downloadable artifact
</code></pre>
<p>This keeps the public manual honest. If a feature is runtime-evidenced, the
manual can say so. If it is source-evidenced, canary, planned, or
review-needed, the manual should carry that status instead of smoothing it over.</p>
<h2 id="ai-friendly-documentation"><a class="no-underline" href="#ai-friendly-documentation">AI-Friendly Documentation</a></h2>
<p>AI-friendly documents are useful because the project is broad, but they are not
the source of truth by themselves. They should preserve:</p>
<ul>
<li>source path or report path,</li>
<li>generation date or extraction snapshot,</li>
<li>evidence status,</li>
<li>mutation boundary,</li>
<li>review state,</li>
<li>promotion target.</li>
</ul>
<p>AI-friendly summaries are allowed on the website when they help a reader
navigate the system and when they remain traceable back to source, HELP,
SelfDoc, MDO, or manualgen artifacts.</p>
<h2 id="website-generation-rule"><a class="no-underline" href="#website-generation-rule">Website Generation Rule</a></h2>
<p><img src="/images/sdlc/parallel-lanes-visual-curation-alt.svg" alt="Alternate parallel lane view for attaching visuals, curating narrative, preserving provenance, and promoting reviewed material"></p>
<p>The website should increasingly be maintained by vertical content generation:</p>
<pre><code class="language-text">D:\code\ccode implementation truth
-> SelfDoc / MDO / manualgen / AI-friendly reviewed artifacts
-> D:\dev\x64base-site public derivatives
-> x64base.com published documentation
</code></pre>
