<h2 id="current-status"><a class="no-underline" href="#current-status">Current status</a></h2>
<p>DotTalk++ / x64base should be described as a working educational and research runtime in active beta, not as a finished commercial database product.</p>
<p>Current proven or active areas include:</p>
<ul>
<li>DBF-style table runtime, cursor state, records, fields, work areas, and table inspection.</li>
<li>x64-family table and metadata work.</li>
<li>INX, CNX, CDX, and LMDB-backed index work.</li>
<li>Memo subsystem with object-oriented architecture and canary coverage.</li>
<li>DotTalk++ command shell and DotScript automation.</li>
<li>HELP, command reflection, <code>CMDHELPCHK</code>, and metadata congruence work.</li>
<li>Relation-aware browsing through <code>ERSATZ</code> / <code>RBROWSE</code>.</li>
<li>Record and tuple browsing through <code>SMARTBROWSE</code>, <code>ARTBROWSE</code>, <code>SIMPLEBROWSE</code>, and <code>SB</code>.</li>
<li>Data dictionary readers, catalog paths, and object resolver code.</li>
<li>Python binding experiments through <code>pydottalk</code>.</li>
<li>GUI work across wxWidgets, Python/Tk preview tools, and Turbo Vision-style TUI surfaces.</li>
<li>GUI/front-end branding now uses <code>ArcticTalk</code> as the public umbrella while
keeping <code>Foxtalk</code> lineage names where they still explain the subsystem story.</li>
</ul>
<p>See <a href="/docs/engine/feature-crosswalk">Engine Feature Crosswalk</a> for the current evidence-backed hierarchy and <a href="/docs/dev/current-lanes">Current Work Lanes</a> for the non-front-page lane inventory.</p>
<h2 id="build-configuration-truth"><a class="no-underline" href="#build-configuration-truth">Build configuration truth</a></h2>
<p>The top-level CMake options define the real build surface:</p>
<pre><code class="language-text">DOTTALK_WITH_TV        ON by default
DOTTALK_WITH_GUI       OFF by default
DOTTALK_WITH_WX        OFF by default
DOTTALK_TV_TESTS       ON by default
DOTTALK_WITH_INDEX     ON by default
DOTTALK_WITH_EDUCATION ON by default
BUILD_PYDOTTALK        OFF by default
DOTTALK_WITH_RELATIONS ON by default
DOTTALK_PROFILE        DEV or PROD
</code></pre>
<p>The build also wires <code>metacollect</code> as a developer executable for low-level metadata and reflection collection. Its CMake comment explicitly says it must not rebuild HELP data, mutate <code>CMDHELPCHK</code>, write DBFs, or mutate publication artifacts.</p>
<h2 id="command-surface-truth"><a class="no-underline" href="#command-surface-truth">Command surface truth</a></h2>
<p>The command reference shows active command surfaces including:</p>
<ul>
<li><code>BUILDLMDB</code> for materializing LMDB-backed tag databases from current table data.</li>
<li><code>CMDHELPCHK</code> for reflection and HELP/catalog validation.</li>
<li><code>ERSATZ</code> and <code>RBROWSE</code> for relation-aware browsing.</li>
<li><code>SIMPLEBROWSE</code> / <code>SB</code> and <code>SMARTBROWSE</code> / <code>ARTBROWSE</code> for tuple and record browsing.</li>
<li><code>WORKSPACE</code> commands for opening, saving, loading, and closing workspaces.</li>
<li><code>DDL</code> commands for schema-oriented DBF creation and validation work.</li>
<li><code>DRAWIO</code> helpers for opening diagram tooling, with metadata-generated diagrams still a future direction.</li>
</ul>
<h2 id="website-wording-rule"><a class="no-underline" href="#website-wording-rule">Website wording rule</a></h2>
<p>Use these words carefully:</p>
<ul>
<li>Use "active beta", "runtime-proven", "active", "partial", "experimental", and "canary" when that is the observed state.</li>
<li>Avoid presenting x64base as complete, commercial, or production-finished.</li>
<li>Avoid claiming DBF_64 and FPT64 are the only storage truth. The local tree contains DBF-style runtime work, x32 and x64-family work, memo systems, xindex backends, data dictionary code, and metadata catalogs.</li>
<li>Treat DotTalk++ as the canonical runtime and teaching shell, not merely a small scripting add-on.</li>
<li>Treat the website as a reviewed derivative of SelfDoc/contracts/manual work,
not as an independent source of technical truth.</li>
