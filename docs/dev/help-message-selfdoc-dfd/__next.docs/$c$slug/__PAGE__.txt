<h2 id="context-dfd"><a class="no-underline" href="#context-dfd">Context DFD</a></h2>
<p><a href="/images/sdlc/help-message-selfdoc-context-dfd-v1.svg"><img src="/images/sdlc/help-message-selfdoc-context-dfd-v1.svg" alt="Context data-flow diagram for the DotTalk++ HELP, message, metadata, SelfDoc, MDO, manual, and publication system"></a></p>
<p>External tools and AI assistants may submit candidate findings. They do not
become authority without source, runtime, evidence, and human review gates.</p>
<h2 id="level-1-complete-system-flow"><a class="no-underline" href="#level-1-complete-system-flow">Level 1: Complete System Flow</a></h2>
<p><a href="/images/sdlc/help-message-selfdoc-level1-dfd-v1.svg"><img src="/images/sdlc/help-message-selfdoc-level1-dfd-v1.svg" alt="Level 1 data-flow diagram showing source, compiled messages, runtime message stores, HELP DATA, metadata, validation, SelfDoc, proof, MDO, manualgen, and website publication"></a></p>
<p>The main stores remain deliberately separate:</p>









































<table><thead><tr><th>Store family</th><th>Role</th></tr></thead><tbody><tr><td>Source and registries</td><td>Implementation, command/function identity, comments, usage, and contracts</td></tr><tr><td>Compiled fallback messages and reference headers</td><td>Bootstrap-safe generated runtime/reference artifacts</td></tr><tr><td>Active messaging workspace</td><td>Locale-aware <code>SYSTEM_MESSAGES</code> and <code>SYSTEM_MESSAGE_TEXT</code> DBF/DTX data plus CDX/LMDB indexes</td></tr><tr><td>HELP DATA v2</td><td>Topics, sections, lines, and typed artifacts used for organized explanation</td></tr><tr><td>Metadata catalogs</td><td><code>SYSCMD</code>, <code>SYSSUBCMD</code>, <code>SYSFUNC</code>, <code>SYSARGS</code>, and <code>SYSMSG</code> normalization and crosswalk data</td></tr><tr><td>SelfDoc and proof</td><td>Manifests, policies, reports, transcripts, hashes, screenshots, and canaries</td></tr><tr><td>MDO and manualgen</td><td>Work orders, decisions, maps, validation reports, manifests, and draft workspaces</td></tr><tr><td>Publication</td><td>Reviewed manual sections, canonical diagram sources, website source, artifact manifests, and versioned public output</td></tr></tbody></table>
<h2 id="level-2-runtime-messages-help-and-metadata"><a class="no-underline" href="#level-2-runtime-messages-help-and-metadata">Level 2: Runtime Messages, HELP, and Metadata</a></h2>
<p><a href="/images/sdlc/help-message-selfdoc-level2-runtime-dfd-v1.svg"><img src="/images/sdlc/help-message-selfdoc-level2-runtime-dfd-v1.svg" alt="Level 2 DFD for source contracts, compiled fallback messages, runtime locale catalogs, HELP DATA assembly, metadata, and CMDHELPCHK"></a></p>
<p>The runtime message resolver follows an override/fallback model:</p>
<ol>
<li>keep compiled fallback text available for startup and recovery;</li>
<li>load the active messaging workspace when reviewed DBF/DTX artifacts exist;</li>
<li>resolve by message symbol and locale;</li>
<li>apply locale fallback and message variables;</li>
<li>emit runtime output without making HELP DATA a startup dependency.</li>
</ol>
<p>HELP DATA is assembled from source contracts, registries, stable message
symbols, compatibility references, metadata, and curated prose. Raw runtime
strings are not mined into HELP DATA by default. <code>CMDHELPCHK</code> compares identity
and coverage across these stores and emits drift evidence rather than silently
repairing authority surfaces.</p>
<h2 id="level-2-selfdoc-proof-mdo-and-publication"><a class="no-underline" href="#level-2-selfdoc-proof-mdo-and-publication">Level 2: SelfDoc, Proof, MDO, and Publication</a></h2>
<p><a href="/images/sdlc/help-message-selfdoc-level2-promotion-dfd-v1.svg"><img src="/images/sdlc/help-message-selfdoc-level2-promotion-dfd-v1.svg" alt="Level 2 DFD for report-only SelfDoc probes, proof capture, MDO review, manualgen validation, diagram attachment, and website publication"></a></p>
<p>The promotion side is conservative:</p>
<ul>
<li>collectors and probes are report-only by default;</li>
<li>scanner confidence and status labels survive curation;</li>
<li>runtime transcripts and screenshots are evidence, not automatic claims;</li>
<li>MDO records accept, reject, defer, and relabel decisions;</li>
<li>manualgen inventories, validates, checks parity, and builds draft workspaces;</li>
<li>human review authorizes narrative and diagram promotion;</li>
<li>public-content, provenance, route, and build guards run before publication;</li>
<li>website technical prose cannot flow backward into the manual by default.</li>
</ul>
