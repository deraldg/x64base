<h2 id="proof-rules"><a class="no-underline" href="#proof-rules">Proof Rules</a></h2>













































<table><thead><tr><th>Status</th><th>Meaning</th><th>Promotion Rule</th></tr></thead><tbody><tr><td>runtime-evidenced</td><td>Source and runtime output or screenshots show the behavior.</td><td>Can be described as current behavior with conservative wording.</td></tr><tr><td>source-evidenced</td><td>Source, headers, registries, or contracts show the behavior.</td><td>Can be documented as implemented/source-defined, but should avoid runtime claims until tested.</td></tr><tr><td>help-catalog-evidenced</td><td>HELP, CMDHELP, command catalog, or metadata lists the surface.</td><td>Can seed catalog pages, then requires source/runtime review.</td></tr><tr><td>generated-reviewed</td><td>A report, matrix, manual section, or diagram was generated and reviewed before publication.</td><td>Can be promoted to website/manual copy with provenance.</td></tr><tr><td>manual-reviewed</td><td>A manual or reference artifact passed its documented review and acceptance gate.</td><td>Can be published with its pinned source snapshot, counts, and regeneration boundary.</td></tr><tr><td>planned-or-in-progress</td><td>The lane is real planning or active development, but not complete.</td><td>Keep under development docs and avoid product-complete wording.</td></tr><tr><td>review-needed</td><td>A scanner, memory note, or draft found something not yet verified.</td><td>Do not promote as a stable website claim.</td></tr></tbody></table>
