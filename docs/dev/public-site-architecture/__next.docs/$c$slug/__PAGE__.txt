<h2 id="x64basecom"><a class="no-underline" href="#x64basecom">x64base.com</a></h2>
<p><a href="/images/sdlc/x64base-com-site-architecture-v1.svg"><img src="/images/sdlc/x64base-com-site-architecture-v1.svg" alt="Architecture of x64base.com from implementation evidence and Laboratory Campus proof through the website source tree, public sections, publication gates, GitHub Pages, and the live main ecosystem website"></a></p>
<p><code>x64base.com</code> is the main ecosystem and engine website. Its public surface owns:</p>
<ul>
<li>engine, DBF-family, memo, index, relation, and API documentation;</li>
<li>DotTalk++, DotScript, workbench, and related product framing;</li>
<li>Laboratory Campus, cases, education, evidence, and collaboration entry points;</li>
<li>project truth, development lanes, standards, DFDs, and publication governance;</li>
<li>selected public media, evidence summaries, and curated download routes.</li>
</ul>
<p>Its technical claims derive from implementation, runtime, HELP, metadata,
contracts, proof, SelfDoc, MDO/manualgen, and reviewed manual/diagram evidence.
Its maintained website source tree produces a versioned GitHub Pages publication
in <code>deraldg/x64base</code>.</p>
<h2 id="dottalkppcom"><a class="no-underline" href="#dottalkppcom">dottalkpp.com</a></h2>
<p><a href="/images/sdlc/dottalkpp-com-site-architecture-v1.svg"><img src="/images/sdlc/dottalkpp-com-site-architecture-v1.svg" alt="Architecture of dottalkpp.com from DotTalk++ evidence, MDO and manualgen, and separately maintained public artifacts through focused technical rooms, publication gates, GitHub Actions, and GitHub Pages"></a></p>
<p><code>dottalkpp.com</code> is the focused technical library and artifact room. Its public
surface owns:</p>
<ul>
<li>curated and versioned manuals;</li>
<li>the DotScript language guide;</li>
<li>command, function, mutator, format, and HELP references;</li>
<li>generated SelfDoc, contract, proof, and manualgen reports;</li>
<li>reviewed downloads, proof packets, accessibility companions, and archives;</li>
<li>source, artifact, README-preservation, and publication governance.</li>
</ul>
<p>Its maintained source tree is versioned in <code>deraldg/dottalkpp</code>, built by GitHub
Actions, and published through GitHub Pages to <code>dottalkpp.com</code>.</p>
<h2 id="connections-between-the-sites"><a class="no-underline" href="#connections-between-the-sites">Connections Between The Sites</a></h2>
<p><a href="/images/sdlc/x64base-dottalkpp-public-sites-connection-v1.svg"><img src="/images/sdlc/x64base-dottalkpp-public-sites-connection-v1.svg" alt="Governed connection map showing the shared upstream authority, independent x64base.com and dottalkpp.com publication lanes, reciprocal reader links, reviewed artifact notices, and blocked reverse authority"></a></p>













































<table><thead><tr><th>Connection</th><th>Direction</th><th>Rule</th></tr></thead><tbody><tr><td>Technical evidence</td><td>Upstream to both sites</td><td>Source/runtime/HELP/metadata/contracts/proof remain authoritative.</td></tr><tr><td>Ecosystem navigation</td><td><code>x64base.com -> dottalkpp.com</code></td><td>Send readers to deeper manuals, references, generated reports, and artifacts.</td></tr><tr><td>Project context</td><td><code>dottalkpp.com -> x64base.com</code></td><td>Send readers to engine, products, Laboratory Campus, history, and governance context.</td></tr><tr><td>Artifact notices</td><td>Reviewed duplex</td><td>dottalkpp.com may contribute reviewed route summaries and downloadable-artifact notices to x64base.com.</td></tr><tr><td>Manual/website reverse flow</td><td>Blocked by default</td><td>Public prose does not become runtime or manual truth.</td></tr><tr><td>Public-only exception</td><td>Review gate</td><td>A separately owned, non-derivable artifact may flow upstream only with provenance.</td></tr><tr><td>Release history</td><td>Independent</td><td>Each site keeps its own source tree, build, routes, and version history.</td></tr></tbody></table>
<p>The shared rule is simple: <strong>one evidence spine, two publication roles</strong>.
x64base.com explains the ecosystem; dottalkpp.com carries the deeper technical
library. Neither site replaces source or runtime authority.</p>
