1d:T247b,<p>This page starts the source museum for the x64base / DotTalk++ line. It does
not replace Git history and it does not make product claims by itself. It
preserves evidence about code families, dates, checksums, verification level,
and the difference between original source, recovered builds, AI-assisted false
starts, and the modern DotTalk++ implementation.</p>
<p>The public rule is simple: <strong>originals stay preserved, analysis stays labeled,
and current product claims must be backed by modern source or runtime proof.</strong></p>
<h2 id="why-this-matters"><a class="no-underline" href="#why-this-matters">Why this matters</a></h2>
<p>The older xBase code is compact and, in some places, more complete than the
recent AI-assisted recovery attempts. The recovered <code>xbase2</code> line contains real
index work. The later buildable DotTalk recovery folders contain useful command
and DBF shell evidence, but their index layer is stubbed. The modern DotTalk++
tree is much larger in scope: DBF flavors, DBF_64, VFP compatibility, LMDB/CDX
indexing, table buffering, dirty/stale state, GUI/TUI workbench surfaces,
SelfDoc, manualgen, and the Laboratory Campus documentation system.</p>
<p>That contrast is important enough to preserve as a first-class documentation
lane.</p>
<h2 id="preservation-layout"><a class="no-underline" href="#preservation-layout">Preservation layout</a></h2>
<p>Use a small archive structure rather than a loose pile of folders:</p>
<pre><code class="language-text">archive/source-lineage/
  1993-xbase/
    original/
    analysis/
    MANIFEST.json
    NOTES.md
  1993-1996-xdll/
    original/
    analysis/
    MANIFEST.json
    NOTES.md
  1995-xbase2-dottalk/
    original/
    buildable-msvc/
    analysis/
    MANIFEST.json
    NOTES.md
  2025-dottalk-recovery/
    original/
    ai-assisted/
    analysis/
    MANIFEST.json
    NOTES.md
  2025-ai-assisted-false-starts/
    gemini/
    grok/
    chatgpt/
    copilot/
    unknown/
    MANIFEST.json
    NOTES.md
  2026-dottalkpp-modern/
    references/
    analysis/
    MANIFEST.json
    NOTES.md
</code></pre>
<p>Do not edit files under <code>original/</code>. Put build fixes, adapters, and notes in
side folders.</p>

<h2 id="index-lineage-correction"><a class="no-underline" href="#index-lineage-correction">Index lineage correction</a></h2>
<p>The recovered DotTalk folders under <code>C:\Users\deral\DotTalk</code> and <code>D:\code\gTalk</code>
have index vocabulary but stubbed index behavior. However, <code>D:\code\xbase2</code>
contains real historical index work:</p>
<ul>
<li><code>D:\code\xbase2\XBINDEX.C</code> implements block-oriented index routines such as
<code>make_index</code>, <code>first_key</code>, <code>next_key</code>, <code>find_key</code>, <code>add_key</code>, <code>openix</code>,
<code>creatix</code>, block retrieval, free-list handling, insertion, split, and search
helpers.</li>
<li><code>D:\code\xbase2\XBCMDS.C</code> wires that index engine into <code>open_index</code>,
<code>build_index</code>, <code>find</code>, and relation positioning.</li>
</ul>
<p>Website and manual copy should say the older index engine exists in the
historical xBase2/XDLL line, while later AI/recovery-era DotTalk copies often
stubbed it out.</p>
<h2 id="ai-assisted-artifacts"><a class="no-underline" href="#ai-assisted-artifacts">AI-assisted artifacts</a></h2>
<p>The AI-assisted folders are recent but often primitive relative to both the
original C code and the modern DotTalk++ tree. They are still valuable because
they show recovery attempts, naming drift, build shims, generated wrappers,
GUI/TUI experiments, and false starts.</p>
<p>Use these labels:</p>

































<table><thead><tr><th>Label</th><th>Meaning</th></tr></thead><tbody><tr><td><code>ai-assisted-experiment</code></td><td>Generated or heavily AI-shaped artifact; useful evidence, not product truth.</td></tr><tr><td><code>stubbed</code></td><td>Command or file exists but core behavior is missing.</td></tr><tr><td><code>superseded</code></td><td>Replaced by a stronger current implementation.</td></tr><tr><td><code>source-reviewed</code></td><td>A human reviewed source text enough to describe it.</td></tr><tr><td><code>build-verified</code></td><td>The artifact was compiled in a named environment.</td></tr><tr><td><code>runtime-verified</code></td><td>A command or feature was executed and captured.</td></tr></tbody></table>

<ul>

</ul>
<p>This seed should be expanded by a report-only script that scans known folders,

<h2 id="website-presentation"><a class="no-underline" href="#website-presentation">Website presentation</a></h2>

can link to the broader documentation summary, but the detailed history should

<p>Public wording should be conservative:</p>
<blockquote>
<p>x64base/DotTalk++ descends from historical xBase/DotTalk experiments and
recovered C-era source. The archive includes original code, modern recovery
work, and AI-assisted false starts. Each artifact is labeled by verification
level so documentation reflects actual behavior, not aspiration.</p>
