6:T13cc,<h2 id="one-system-cooperating-projects"><a class="no-underline" href="#one-system-cooperating-projects">One system, cooperating projects</a></h2>
<p>x64base is developed through cooperating systems rather than a one-way sequence
from code to documentation. Runtime behavior produces evidence; metadata and
SelfDoc organize it; the Laboratory Campus turns reviewed evidence into labs,
cases, diagrams, and lessons; public documentation exposes omissions and
contradictions that return to implementation work.</p>
<p>This is <strong>recursion</strong> in the engineering sense: the outputs of one governed pass
become inputs to the next pass through the system.</p>
<h2 id="three-primary-coproject-systems"><a class="no-underline" href="#three-primary-coproject-systems">Three primary coproject systems</a></h2>
<h3 id="1-x64base-and-dottalk-implementation"><a class="no-underline" href="#1-x64base-and-dottalk-implementation">1. x64base and DotTalk++ implementation</a></h3>
<p>The engine, command runtime, DotScript, contracts, HELP surfaces, and executable
behavior live in the implementation repository. That source and its captured
runtime evidence remain the primary technical truth.</p>
<h3 id="2-metadata-selfdoc-and-laboratory-campus"><a class="no-underline" href="#2-metadata-selfdoc-and-laboratory-campus">2. Metadata, SelfDoc, and Laboratory Campus</a></h3>
<p>Metadata and SelfDoc collect, validate, organize, and preserve provenance.
Laboratory Campus, with LabTalk retained as its project and historical name,
turns reviewed material into diagrams, datasets, cases, demonstrations, labs,
and lessons.</p>
<h3 id="3-public-documentation-systems"><a class="no-underline" href="#3-public-documentation-systems">3. Public documentation systems</a></h3>
<p>Generated manuals, x64base.com, and the supporting DotTalk++ artifact site
publish reviewed derivatives. They are also diagnostic surfaces: organizing the
system for readers reveals missing contracts, stale HELP, contradictory claims,
and absent proof.</p>
<h2 id="the-recursive-loop"><a class="no-underline" href="#the-recursive-loop">The recursive loop</a></h2>
<pre><code class="language-text">engine behavior
    -> runtime evidence and source contracts
    -> HELP, metadata, and SelfDoc collection
    -> reviewed reports, diagrams, cases, and lessons
    -> manuals and public documentation
    -> review reveals gaps, drift, and contradictions
    -> findings return to metadata and implementation
    -> the next pass produces stronger evidence
</code></pre>
<p>The loop is recursive, but not circular authority. Public prose does not become
engine truth merely because it is published. Generated material cannot promote
itself. Provenance, status labels, and review gates control every return path.</p>
<h2 id="labtalk-is-broader-than-an-executable"><a class="no-underline" href="#labtalk-is-broader-than-an-executable">LabTalk is broader than an executable</a></h2>
<p>Laboratory Campus is a hybrid collaboration layer containing implemented
components, developing labs, datasets, runtime demonstrations, help systems,
GUI and portal experiments, planning material, and proof artifacts. An
incomplete public repository boundary does not make the working local coproject
empty.</p>
<h2 id="governing-doctrine"><a class="no-underline" href="#governing-doctrine">Governing doctrine</a></h2>
<blockquote>
<p><strong>Runtime proves, source defines, HELP explains, metadata organizes, and
SelfDoc preserves.</strong></p>
</blockquote>
<p>The website publishes reviewed derivatives of that chain. Its review then
recurses into the next engineering pass, making documentation an active
instrument of development rather than an after-the-fact report.</p>
<h2 id="authority-and-safeguards"><a class="no-underline" href="#authority-and-safeguards">Authority and safeguards</a></h2>
<ul>
<li>Runtime and source remain authoritative for implementation claims.</li>
<li>SelfDoc and metadata preserve provenance rather than manufacturing proof.</li>
<li>Existing, developing, planned, and experimental work must retain honest
status labels.</li>
<li>Website review may identify a source defect; changing public prose alone does
not repair it.</li>
<li>Manual and website exchange follows governed simplex and duplex gates.</li>
<li>Canonical <code>README.*</code> files are preserved and versioned when replaced.</li>
</ul>
