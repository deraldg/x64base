<h2 id="ownership"><a class="no-underline" href="#ownership">Ownership</a></h2>
<p>DotTalk++ SDLC owns:</p>
<ul>
<li>runtime behavior,</li>
<li>command implementation,</li>
<li>storage and indexing behavior,</li>
<li>HELP / CMDHELP / CMDHELPCHK alignment,</li>
<li>build and smoke proof,</li>
<li>maintenance surfaces such as MAINT, BBOX, DDICT, and MANUAL.</li>
</ul>
<p>LabTalk and the Laboratory Campus may teach DotTalk++ behavior, but DotTalk++
owns whether that behavior is real.</p>
<h2 id="evidence-ladder"><a class="no-underline" href="#evidence-ladder">Evidence Ladder</a></h2>
<p><img src="/images/sdlc/dottalk-runtime-promotion-path.svg" alt="DotTalk++ runtime promotion path"></p>
<pre><code class="language-text">source_defined
-> runtime_observed
-> help_documented
-> validated
-> professional_ready
-> lab_ready
</code></pre>
<p>LabTalk should not package DotTalk++ behavior as real until DotTalk++ SDLC has
runtime evidence, and preferably validation for student-facing use.</p>
