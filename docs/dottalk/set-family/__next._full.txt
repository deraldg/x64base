1b:T12e6,<p>The SET family is documented as an engine feature, not just a collection of shell conveniences. <code>src/cli/cmd_set.cpp</code> is the general dispatcher, and specialized <code>cmd_set*.cpp</code> handlers own path, index/order, filter, relation, locale, and unique-state behavior.</p>
<h2 id="dispatcher-boundary"><a class="no-underline" href="#dispatcher-boundary">Dispatcher boundary</a></h2>
<p><code>SET</code> with no arguments reports usage. <code>SET USAGE</code> is read-only. Mutating forms change session/runtime state but should not directly rewrite table records unless the delegated command explicitly performs data work.</p>
<p>Important delegated forms include:</p>
<pre><code class="language-text">SET INDEX ...
SET ORDER ...
SET FILTER ...
SET RELATION ...
SET PATH ...
SET CNX ...
SET CDX ...
SET LMDB ...
SET UNIQUE ...
</code></pre>
<h2 id="output-and-session-presentation"><a class="no-underline" href="#output-and-session-presentation">Output and session presentation</a></h2>
<p>The dispatcher covers command-output behavior:</p>
<pre><code class="language-text">SET CONSOLE ON|OFF
SET PRINT ON|OFF
SET PRINT TO &#x3C;file>
SET DEVICE TO SCREEN|FILE &#x3C;path>|PRINTER|NULL
SET ALTERNATE ON|OFF
SET ALTERNATE TO &#x3C;file>
SET TALK ON|OFF
SET ECHO ON|OFF
SET PAGING ON|OFF
SET WRAP ON|OFF
</code></pre>
<p>The settings registry also includes presentation and runtime flags such as <code>SET STATUS</code>, <code>SET TIME</code>, <code>SET TIMER</code>, <code>SET CLOCK</code>, <code>SET BELL</code>, <code>SET TYPEAHEAD</code>, and <code>SET REPROCESS</code>.</p>
<h2 id="data-and-editing-behavior"><a class="no-underline" href="#data-and-editing-behavior">Data and editing behavior</a></h2>
<p>Current documented surfaces include:</p>
<pre><code class="language-text">SET TABLE BUFFER ON|OFF|ON ALL|OFF ALL
SET DELETED ON|OFF
SET CASE ON|OFF
SET NEAR ON|OFF
SET SAFETY ON|OFF
SET EXACT ON|OFF
SET ESCAPE ON|OFF
SET CARRY ON|OFF
SET CONFIRM ON|OFF
SET EXCLUSIVE ON|OFF
SET MULTILOCKS ON|OFF
</code></pre>
<p><code>SET TABLE BUFFER</code> toggles buffering for the current area or all open areas. <code>COMMIT</code> and <code>ROLLBACK</code> remain the commands that finish or discard buffered edits.</p>
<h2 id="paths"><a class="no-underline" href="#paths">Paths</a></h2>
<p><code>SET PATH</code> and <code>SETPATH</code> route named filesystem slots:</p>
<pre><code class="language-text">DATA
DBF
XDBF
INDEXES
LMDB
WORKSPACES
SCHEMAS
PROJECTS
SCRIPTS
TESTS
HELP
LOGS
TMP
</code></pre>
<p><code>SETPATH RESET</code> returns slots to defaults. Path validation warns when needed, but assignment is still explicit state.</p>
<h2 id="index-and-order"><a class="no-underline" href="#index-and-order">Index and order</a></h2>
<p>The index-related SET family is flavor-aware:</p>




















<table><thead><tr><th>Area flavor</th><th>Default order container</th><th>Accepted SET INDEX containers</th></tr></thead><tbody><tr><td>V32 classic/MS-DOS/FoxPro-like</td><td><code>.cnx</code></td><td><code>.inx</code>, <code>.cnx</code></td></tr><tr><td>V64 VFP/x64</td><td><code>.cdx</code></td><td><code>.cdx</code></td></tr></tbody></table>
<p><code>SET ORDER 0</code>, <code>SET ORDER PHYSICAL</code>, <code>SET ORDER NATURAL</code>, and <code>SET ORDER PHYS</code> clear the active order. <code>SET ORDER TAG &#x3C;tag></code> activates a tag in the flavor-appropriate container. <code>INX</code> remains a single-order <code>SET INDEX</code> attachment path.</p>
<h2 id="filters-and-relations"><a class="no-underline" href="#filters-and-relations">Filters and relations</a></h2>
<p><code>SET FILTER</code> installs the current expression filter used by navigation and list-style commands.</p>
<p><code>SET RELATION</code> manages parent-to-child relation state:</p>
<pre><code class="language-text">SET RELATION TO &#x3C;expr> INTO &#x3C;child>
SET RELATION TO &#x3C;expr> INTO &#x3C;child> ADDITIVE
SET RELATION OFF ALL
SET RELATION OFF INTO &#x3C;child>
</code></pre>
<p>Relation state belongs with areas and workspaces, so documentation should describe it as part of the object-oriented runtime state.</p>
<h2 id="locale-and-formatting"><a class="no-underline" href="#locale-and-formatting">Locale and formatting</a></h2>
<p>The SET family also carries presentation and formatting settings:</p>
<pre><code class="language-text">SET LANGUAGE TO &#x3C;locale|DEFAULT>
SET LOCALE TO &#x3C;locale|DEFAULT>
SET LANGUAGE CHECK
SET LANGUAGE REPORT
SET LOCALE CHECK
SET LOCALE REPORT
SET CENTURY ON|OFF
SET DATE TO &#x3C;format>
SET DECIMALS TO &#x3C;n>
SET FIXED ON|OFF
SET MEMOWIDTH TO &#x3C;n>
SET MEMOERROR ON|OFF
SET EDITOR TO &#x3C;value>|DEFAULT|OFF
</code></pre>
