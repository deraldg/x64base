4:T702,<p>DotTalk++ is the canonical command surface for the local x64base runtime.</p>
<p>It is used for:</p>
<ul>
<li>opening tables and workspaces.</li>
<li>inspecting areas, fields, records, relations, and indexes.</li>
<li>running HELP and validation.</li>
<li>browsing records and relation graphs.</li>
<li>running scripts and smoke/canary workflows.</li>
</ul>
<p>DotScript is the productized script form of this command language. DotTalk++ is the shell and runtime vocabulary; DotScript is the repeatable file language for labs, validation runs, automation, loops, variables, comments, and documented canary scripts. See <a href="/docs/dottalk/dotscript-language-guide">DotScript Language Guide</a>.</p>

<h2 id="status"><a class="no-underline" href="#status">Status</a></h2>
<p>The command language is active and usable, but individual commands vary by proof level. Some are runtime-proven, some are active beta, and some remain canary or integration surfaces.</p>
