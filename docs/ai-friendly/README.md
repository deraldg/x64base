# AI Friendly Lane

Status: seed lane.

## Purpose

AI Friendly is the DotTalk++ / LabTalk lane for turning useful AI-assisted work
into reviewed, anchored, reusable project knowledge.

This lane does not replace SelfDoc, MDO/manualgen, contracts, HELP, CMDHELPCHK,
LabTalk, proof reports, or source comments. It routes material into those
systems and looks for simplification, consolidation, and integration before
creating anything new.

The task-context, typed-synapse, guarded-execution, and x64base self-hosting
work is tracked as a Laboratory Campus lane under `labtalk/ai_portal`. AI
Friendly continues to curate and route AI-assisted knowledge; the AI Portal
hardening lane owns the operational context-assembly system.

## Core Rule

AI interaction material is source material, not authority.

Promote only after it is classified, distilled, anchored to existing evidence,
and routed to the appropriate DotTalk++ or LabTalk lane.
