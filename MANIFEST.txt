DotTalk++ X64 runtime field-length fix drop

Purpose:
  Preserve X64 character fields larger than 255 bytes through CREATE -> disk -> loader -> runtime -> STRUCT/FIELDS.

Replacement paths:
  include/xbase.hpp
  include/xbase_64.hpp
  include/xbase_vfp.hpp
  include/xbase/dbf_create.hpp
  src/xbase/dbf_create.cpp
  src/cli/dbf64_header_validate.cpp
  src/cli/cmd_sort.cpp
  src/core/fields_mgr.cpp

Key contract:
  FieldDef.length is runtime truth and is now uint32_t.
  Classic/VFP descriptor lengths remain uint8_t.
  X64 descriptor length uses 255 as a compatibility marker for extended C fields.
  X64M metadata version 2 stores authoritative field length.
  X64 loader applies metadata length back into DbArea::FieldDef.

Recommended clean test:
  CREATE X64 BIGCHAR_TEST (ID C(16), BIGTEXT C(1024))
  STRUCT

Expected:
  BIGTEXT C 1024
