SYSARGS canonical import proposal v1
===================================

Purpose:
- keep SYSARGS seeding read-only until reviewed
- preserve the same canonical-import pattern used for SYSCMD and SYSFUNC
- source authority is usage-contract extraction from `src/cli` headers

Do not execute this file as-is. Review first.

Proposed sequence
-----------------

1. Open metadata workspace/environment.
2. Recreate or verify the native `SYSARGS` x64 table shape from:
   `scripts/SYSTEM_METADATA_SCHEMA_v4_safe10_FIXED.dts`
3. Open `SYSARGS`.
4. Ensure the x64 CDX container exists.
5. Build LMDB after import if desired.
6. Import:
   `scripts/metadata/SYSARGS_IMPORT_v1.csv`

Reference DotScript sketch
--------------------------

DO metadata
WORKSPACE LOAD metadata

SELECT SYSARGS
ZAP

IMPORT scripts\\metadata\\SYSARGS_IMPORT_v1.csv

SET INDEX TO SYSARGS
BUILDLMDB CLEAN YES

Notes
-----

- `SYSARGS_IMPORT_v1.csv` is generated by `metacollect`.
- Canonical export currently focuses on value-bearing placeholder arguments such as:
  - `TABLE`
  - `FIELD`
  - `VALUE_EXPR`
  - `PRED`
  - `COUNT`
- Dev-only command families may be excluded from canonical export even if they are
  still registered in the runtime. Current example: `BETA`.
- Literal keyword rows are available as an opt-in diagnostic export later if needed,
  but are not part of the canonical seed by default.
- Required/repeat flags are inferred conservatively from usage variants.
- Locale/region seams are present now:
  - `DEF_LOCALE = en-US`
  - `REGION_ID = GLOBAL`
