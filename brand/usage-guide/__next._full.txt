13:T537,<h2 id="naming"><a class="no-underline" href="#naming">Naming</a></h2>
<ul>
<li>Use <strong>x64base®</strong> for the platform name.</li>
<li>Use <strong>DBF_64</strong> and <strong>FPT64</strong> exactly as written.</li>
<li>Use product names consistently (DotTalk++, DotTalk++ Workbench, TupTalk, TableTalk, RelTalk, Parallel GUI/TUI, LabTalk).</li>
<li>Use <strong>DotTalk++ Workbench</strong> as the family name for GUI, TUI, browser, and workbench surfaces.</li>
<li>Use <strong>Arctic</strong> as the code name for the TUI surface, not as the family umbrella or a main product.</li>
<li>Use <strong>TableTalk</strong> for table buffering, dirty/stale state, and commit/rollback workflows, not generic table browsing.</li>
<li>Keep <strong>Foxtalk</strong> and related fox-derived names for lineage/subsystem references only where the source/evidence supports them.</li>
<li>Preserve classic FoxPro-style syntax/history where true, but do not describe DotTalk++ as a clone.</li>
</ul>

</ul>
